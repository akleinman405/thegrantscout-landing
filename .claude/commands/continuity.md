# Create a Session State File (Handoff Document)

This command WRITES a new session state file. It does NOT read or load existing session files.

**This is NOT the /resume command.** Do NOT search for or read existing SESSION_STATE or CONTINUITY files. Your job is to CREATE a new one based on the current conversation and (optionally) git state.

Task description: $ARGUMENTS

## Instructions

**STEP 0: Validate arguments.** If `$ARGUMENTS` is empty or blank, ask the user: "What task should I capture session state for?" Do NOT proceed until you have a task description.

**STEP 1: Review this conversation** (the messages above in this chat session) to extract:
- Decisions made during this session (with rationale)
- Files created, modified, or deleted
- Database changes (tables created, data loaded, queries that worked)
- Approaches that were tried and failed (and why)
- What's working vs. what's still broken
- Any partially implemented features
- Key data findings or insights
- What the user should do next

This is the **primary data source**. Even sessions with zero git activity produce valuable state.

If this is a fresh conversation with no prior messages, skip this step and note "No conversation context available" in the Decisions section.

**STEP 2: Gather git + filesystem state (supplementary).** Run these in parallel:
- `git diff` to see unstaged changes
- `git diff --cached` to see staged changes
- `git status` to list modified/new/deleted files
- `git log --oneline -5` for recent commits

If git shows no changes, that's fine. Note "No git activity this session" in the Files section and move on. Git is supplementary context, not required.

**STEP 3: Write session state to memory directory.** Write the file to:
`~/.claude/projects/-Users-aleckleinman-Documents-TheGrantScout/memory/session_state.md`

This is the **primary output** that `/resume` will find. Use this template:

```markdown
# SESSION STATE: [Task Description from $ARGUMENTS]

**Last Updated:** YYYY-MM-DD HH:MM
**Working Folder:** [primary folder where work happened, e.g. 5. Runs/VetsBoats/2026-02-06/]
**Status:** In Progress / Blocked / Ready for Review

---

## RESUME POINT (Read This First)

### Do This Next
[Exactly what the next session should do first. Be specific: file names, function names, line numbers, SQL queries.]

### Remaining Steps
1. [Most important remaining item]
2. [Second priority]
3. [Third priority]

---

## What's Done
- [Completed items with specifics]

## What's Partially Done
- [In-progress items: what works, what doesn't yet]

## What's Not Started
- [Remaining items if known]

---

## Decisions Made

| Decision | Alternatives Considered | Rationale |
|----------|------------------------|-----------|
| [decision] | [what else was considered] | [why this choice] |

---

## What Was Tried (Failed Approaches)

| Approach | Why It Failed | Lesson |
|----------|--------------|--------|
| [approach] | [what went wrong] | [what to do instead] |

---

## Files Created/Modified

| File | Status | Description |
|------|--------|-------------|
| `path/to/file` | Created/Modified/Deleted | [what changed] |

---

## Database Changes Made

[Tables created, data loaded, schema changes, or "No database changes this session"]

---

## Reports Created

| File | Description |
|------|-------------|
| `REPORT_YYYY-MM-DD.N_description.md` | [what it covers] |

---

## Database Queries That Worked

[Include useful queries discovered during this session, or "None"]

---

## Key Data Findings

[Important numbers, patterns, or insights discovered during this session]

---

## Blockers (if any)

| Blocker | Severity | Workaround |
|---------|----------|------------|
| [issue] | HIGH/MED/LOW | [possible workaround or "None"] |

---

*Generated by Claude Code on YYYY-MM-DD HH:MM*
```

**STEP 4: Write archival copy to project folder.** Determine the appropriate folder:
- Pipeline/enhancement work → `4. Pipeline/Enhancements/YYYY-MM-DD/`
- Client run work → `5. Runs/{Client}/YYYY-MM-DD/`
- Business/sales work → `6. Business/`
- If unclear, use `4. Pipeline/Enhancements/YYYY-MM-DD/`

Write the same content as `SESSION_STATE_YYYY-MM-DD_description.md` (create directory if needed, use snake_case description from $ARGUMENTS).

**STEP 5: Update MEMORY.md with session pointer.** Edit `~/.claude/projects/-Users-aleckleinman-Documents-TheGrantScout/memory/MEMORY.md` to add or replace a "Last Session" block at the very top (before the `## Key Learnings` heading):

```markdown
## Last Session
- **Date:** YYYY-MM-DD
- **Task:** [brief task description from $ARGUMENTS]
- **Status:** [In Progress / Blocked / Complete]
- **Resume:** [one-line description of what to do next]
- **Archive:** [path to archival copy in project folder]
```

If a "Last Session" block already exists, replace it entirely with the new one.

**STEP 6: Confirm** by showing the user:
- Memory file path (`memory/session_state.md`)
- Archival copy path
- Brief summary of what was captured (accomplishments, next step, file count)
