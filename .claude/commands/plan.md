2. **Research the codebase** by searching for relevant patterns:

3. **Check known pitfall areas** — flag any that apply to this task:
   - **Schema prefix**: All queries must use `f990_2025.table_name` — missing prefix causes silent wrong-table reads
   - **EIN handling**: EINs are VARCHAR — must preserve leading zeros, no dashes, never cast to integer
   - **Database name**: Must connect to `thegrantscout` (NOT `postgres`)
   - **fact_grants joins**: Use `foundation_ein` (NOT `filer_ein`) when joining fact_grants
   - **Data quality filters**: `only_contri_to_preselected_ind = FALSE` means open to applications; filter website junk values ("N/A", "NONE", "0")
   - **Temporal analysis**: Use `tax_year` not grant dates (grant dates are often NULL)
   - **File naming**: Must follow `DOCTYPE_YYYY-MM-DD.N_description.ext` convention
   - **Excel formatting**: Must follow standards (blank row 1/col A, zoom 140%, TableStyleMedium2, Calibri 10pt, etc.)
   - **Report structure**: Must include TOC, Revision History, Executive Summary per CLAUDE.md
   - **Client known funders**: Must exclude from results — check `dim_clients.known_funders`
   - **Production filters**: assets >= $100K, unique_recipients >= 5, preselected_only = FALSE
   - **Python/pip**: Use `python3` and `pip3` on macOS (not `python`/`pip`)
