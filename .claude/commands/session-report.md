# Generate Daily Session Report

Auto-generate the daily SESSION_REPORT by scanning git history, individual REPORT files, and current state.

## Instructions

1. **Determine today's date** in YYYY-MM-DD format.

2. **Gather data** by running these in parallel:
   - `git log --since="today 00:00" --oneline --no-decorate` to get today's commits
   - `git status` to check for uncommitted changes
   - Read all files matching `4. Pipeline/Enhancements/YYYY-MM-DD/REPORT_*.md` for today's date
   - Read all files matching `4. Pipeline/Enhancements/YYYY-MM-DD/SESSION_STATE_*.md` for today's date
   - Read all files matching `5. Runs/*/YYYY-MM-DD/REPORT_*.md` for today's date (client work)
   - `git diff --stat` to see changed files summary

3. **Check for existing SESSION_REPORT**: If `4. Pipeline/Enhancements/YYYY-MM-DD/SESSION_REPORT_YYYY-MM-DD.md` already exists, read it and update it rather than replacing from scratch.

4. **Synthesize the SESSION_REPORT** using this exact template structure:

```markdown
# SESSION REPORT: YYYY-MM-DD

**Date:** YYYY-MM-DD
**Status:** Complete / In Progress
**Last Updated:** YYYY-MM-DD

---

## Resume Point (CRITICAL - Read First)

**Priority 1:** [Most urgent next action]
- Context: [Brief explanation]
- Last attempted: [What was tried]

**Priority 2:** [Second priority item]

**Uncommitted Changes:**
- `file.ext` - [description]

---
```

5. **Before writing**, present the **Resume Point** section to the user and ask them to confirm or edit it — this requires human judgment about what's most important to pick up next.

6. **Write the file** to `4. Pipeline/Enhancements/YYYY-MM-DD/SESSION_REPORT_YYYY-MM-DD.md` (create the directory if it doesn't exist).

7. **Confirm** by showing the user the file path and a summary of what was captured (commit count, report count, any blockers flagged).
