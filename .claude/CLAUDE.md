## Project Overview

**TheGrantScout** helps nonprofits find foundations most likely to fund them. We surface funders they didn't know about and provide intelligence to maximize success.

**Core Promise:** "Your mission deserves funding. We'll help you find it."

## Critical Rules

**IMPORTANT - Database:**
- Always use schema prefix: `f990_2025.table_name`
- Database name: `thegrantscout` (NOT postgres)
- Use `foundation_ein` (not `filer_ein`) in fact_grants joins
- EINs are VARCHAR - preserve leading zeros, no dashes

**IMPORTANT - Data Quality:**
- `only_contri_to_preselected_ind = FALSE` = open to applications
- Filter website fields: exclude "N/A", "NONE", "0" values
- Use `tax_year` for temporal analysis (grant dates often NULL)

**IMPORTANT - Files:**
- Follow naming: `DOCTYPE_YYYY-MM-DD.N_description.ext` (N = prompt sequence number)
- Every output must be traceable to a prompt (see Document Naming Convention below)
- Output files go in same folder as prompt unless specified
- Response length: brief unless asked for detail

**IMPORTANT - Excel Files:**
When generating Excel files, follow these standards:
- **Layout:** Blank row 1 and column A (data starts at B2)
- **Zoom:** Set to 140% (`ws.sheet_view.zoomScale = 140`)
- **Table:** Use Excel Table for filtering/sorting (`openpyxl.worksheet.table.Table`)
- **Numbers:** Format with commas (`#,##0`)
- **Currency:** Format as `"$"#,##0`
- **Percentages:** Store as decimal, format as `0%`
- **Style:** Use `TableStyleMedium2` with row stripes
- **Font:** Calibri 10pt
- **Headers:** Bold white text on dark blue (#2F5496)
- **Borders:** Bold (medium) border around table edges and header row; thin borders inside
- **Text:** No wrap (`wrap_text=False`)
- **Freeze panes:** Keep headers and first data column visible

## Gotchas

| Issue | Solution |
|-------|----------|
| XML namespace fails XPath | Use `safe_xpath()` with both namespaced and non-namespaced paths |
| COPY schema error | `SET search_path TO f990_2025` first |
| Duplicate XMLs in ZIPs | Check `stg_processed_xml_files` table, skip duplicates |
| Schema drop danger | Always use `drop_existing=False` for incremental imports |
| Wrong XPath for purpose | Use `GrantOrContributionPurposeTxt` not `PurposeOfGrantTxt` |
| DB connection refused | Check PostgreSQL service is running (`brew services start postgresql`) |
| pip command not found | Use `pip3` instead of `pip` on macOS |
| python command not found | Use `python3` instead of `python` on macOS |

## Foundation Scoring (LASSO V6.1 Model)

Foundations are scored using a **LASSO logistic regression model** with size-matched negative sampling. See `3. Models/v6.1/REPORT_2026-01-03_v6.1_model_evaluation.md` for full details.

| Metric | Value |
|--------|-------|
| Model | LASSO V6.1 (trained 2026-01-03) |
| Test AUC | 0.94 |
| Training Data | 1.31M rows (size-matched negatives) |
| Config | `4. Pipeline/config/coefficients.json` |
| Scorer | `4. Pipeline/scripts/02_score_foundations_v6.1.py` |

### Top Positive Signals (increase score)

| Feature | Coefficient | What It Measures |
|---------|-------------|------------------|
| `match_state_pct` | +1.69 | Foundation % grants in recipient's state |
| `match_sector_pct` | +1.60 | Foundation % grants in recipient's NTEE sector |
| `match_state_x_geo_conc` | +0.54 | State match × geographic focus |
| `f_log_total_grants` | +0.39 | Active foundations |
| `r_total_revenue` | +0.26 | Recipient size (balanced) |
| `match_semantic_similarity` | +0.04 | Mission alignment via embeddings (requires regeneration) |

### Production Filters

| Filter | Effect | Foundations |
|--------|--------|-------------|
| has_grant_history | Active funders only | 112,520 |
| assets >= $100K | Meaningful capacity | 17,613 |
| unique_recipients >= 5 | No captive trusts | 13,735 |
| preselected_only = FALSE | Open to applications | **12,653** |

### Key Behaviors

- **Prior funders:** EXCLUDED from results
- **Scoring:** Uses logit (not probability) for ranking to avoid saturation
- **Scaling:** All features z-score scaled before applying coefficients
- **Semantic similarity:** Requires embedding regeneration (tables archived 2026-01-04)

## Document Naming Convention

### Format
`DOCTYPE_YYYY-MM-DD.N_description.ext`

| Component | Description | Example |
|-----------|-------------|---------|
| DOCTYPE | Document type | PROMPT, REPORT, SPEC, etc. |
| YYYY-MM-DD | Date created | 2025-12-22 |
| .N | Prompt sequence number | .1, .2, .3... |
| description | Brief snake_case description | schema_cleanup |
| ext | File extension | .md, .sql, .py, .csv |

### Prompt-Output Tracking

**Every output must be traceable to a prompt.**

1. PROMPT files are numbered sequentially per day: `.1`, `.2`, `.3`
2. All outputs include the prompt number they were generated from
3. Files without dates (e.g., `model.pkl`) get renamed to include date and prompt: `model_2025-12-22.3.pkl`

### Verbal Instructions Rule

**If the user gives instructions directly in Claude Code CLI (not via a file prompt), Claude MUST:**

1. Create a PROMPT file first documenting what was asked
2. Use the next available prompt number for that day
3. Name outputs using that prompt number
