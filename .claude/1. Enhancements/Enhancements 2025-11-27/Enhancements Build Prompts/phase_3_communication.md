# Phase 3: Communication (Handoffs + Mailbox)

## Objective

Improve how agents communicate with each other. Currently:
- Agents pass raw outputs forward (can be 10,000+ tokens)
- Mailbox.jsonl grows unbounded
- No structured handoff between phases

This phase adds:
1. **Phase Summary Protocol** - Each agent writes a 500-word summary for the next agent
2. **Mailbox Rotation** - Archive old events, keep active mailbox bounded
3. **Handoff Template** - Standard format for phase transitions

---

## Context

The current handoff flow:
```
Scout finishes → dumps files in research_outputs/01_scout/
Analyst starts → reads ALL of Scout's files (potentially huge)
```

The improved flow:
```
Scout finishes → writes phase_summary.md (500 words max)
Analyst starts → reads phase_summary.md first, then specific files as needed
```

This prevents context bloat and forces agents to distill key insights.

## Tasks

### 1. Create Phase Summary Template

Create `.claude/templates/phase_summary_template.md`:

```markdown
# Phase Summary: [Phase Name]

**Agent**: [Agent Name]  
**Completed**: [Timestamp]  
**Duration**: [Time spent]

## Key Outputs
- [Bullet list of main deliverables]
- [File paths to important outputs]

## Key Findings
[2-3 sentences: What did we learn? What's most important?]

## For Next Phase
[2-3 sentences: What should the next agent prioritize? Any gotchas?]

## Blockers Encountered
[If any: what went wrong and how it was resolved]

## Metrics
| Metric | Value |
|--------|-------|
| Items Processed | X |
| Success Rate | Y% |
| Quality Score | Z |

---
*Summary generated by [agent] on [date]. Full outputs in [directory].*
```

### 2. Add Handoff Protocol to All Agents

Add this section to EVERY agent .md file (scout.md, analyst.md, builder.md, etc.):

```markdown
## Handoff Protocol

### Before Starting Work
1. Check for `phase_summary.md` from the previous phase
2. Read the summary FIRST before diving into raw files
3. Note any warnings or recommendations from previous agent

### Before Completing Work
1. Create `phase_summary.md` in your output directory
2. Use template from `.claude/templates/phase_summary_template.md`
3. Keep summary under 500 words
4. Include: key outputs, findings, recommendations for next phase
5. Log completion event to mailbox.jsonl

### Summary Requirements
- Maximum 500 words
- Must include "For Next Phase" section
- Must list file paths to key outputs
- Must include metrics if applicable
```

Insert this section AFTER "Core Responsibilities" and BEFORE "Workflow" in each agent file.

### 3. Create Mailbox Rotation Script

Create `.claude/scripts/rotate_mailbox.ps1` (PowerShell for Windows):

**Requirements**:
- Move events older than 7 days to archive
- Keep active mailbox under 500 lines
- Create archive files by month: `mailbox_archive/2025-11.jsonl`
- Preserve all data (never delete, just move)
- Handle empty mailbox gracefully

**Usage**:
```powershell
.\.claude\scripts\rotate_mailbox.ps1
# Archives old events, keeps recent ones
```

**Also create a `.sh` version** for Git Bash/WSL users if Claude can do both.

### 4. Create Mailbox Archive Folder

Create `.claude/state/mailbox_archive/` folder with a `.gitkeep` file.

### 5. Add Rotation Reminder to Project Completion

Update the Reporter agent's completion protocol to include:
```markdown
### On Project Completion
1. Generate final deliverables
2. Create phase_summary.md
3. Log project_complete event to mailbox
4. Run mailbox rotation: `./scripts/rotate_mailbox.sh`
5. Update state.json with completion timestamp
```

## Constraints

- Phase summaries must be under 500 words
- Don't modify the core logic of any agent (only add Handoff Protocol section)
- Mailbox rotation must NEVER delete data (only archive)
- All agents must get the same Handoff Protocol section (consistency)
- Templates should use standard markdown formatting

## Rollback

If something goes wrong:

**PowerShell:**
```powershell
# Restore agents from git
git checkout -- .claude\agents\

# Remove new files
Remove-Item .claude\templates\phase_summary_template.md
Remove-Item .claude\scripts\rotate_mailbox.ps1
Remove-Item -Recurse -Force .claude\state\mailbox_archive\

# Restore mailbox if corrupted
Copy-Item .claude\state\mailbox.jsonl.backup .claude\state\mailbox.jsonl
```

**Or Git Bash:**
```bash
git checkout -- .claude/agents/
rm .claude/templates/phase_summary_template.md
rm .claude/scripts/rotate_mailbox.*
rm -rf .claude/state/mailbox_archive/
cp .claude/state/mailbox.jsonl.backup .claude/state/mailbox.jsonl
```
