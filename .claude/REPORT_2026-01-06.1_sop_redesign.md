## Problems Addressed

### Problem 1: No Client Understanding Phase

**Before:** Jump straight to algorithm, hope it matches client needs.

**After:** Phase 1 requires verifying:
- Customer type (Calendar Manager / Niche Seeker / Relationship Builder)
- Multi-priority clients identified upfront
- Geographic scope (city vs. state vs. national)
- Known funders to exclude

### Problem 2: Algorithm Doesn't Match Client Needs

**Example:** HN is national organization but got only CT foundations.

| Client | Requested | Algorithm Output | Gap |
|--------|-----------|------------------|-----|
| HN | National | CT only | Critical |
| FCSD | San Diego | Aptos, CA (400 miles) | Critical |
| PSMF | Patient safety | Generic health | Moderate |

**After:** SOP now includes:
- Decision tree for when to use algorithm vs. manual curation
- Required quality check of top 20 before proceeding
- Explicit guidance for national orgs, niche geography, niche sectors

### Problem 3: Multi-Priority Clients Get Partial Coverage

**Example:** FCSD wanted bakery funding AND building funding; got only bakery-related foundations.

**After:** New multi-priority handling:
1. Document EACH priority separately
2. Run foundation discovery per priority
3. Merge results with minimum 2 foundations per priority
4. Label each foundation's relevance in report

### Problem 4: No Quality Checkpoints

**Before:** Report generated and delivered with no systematic quality review.

**After:** Phase 5 introduces:
- Pre-delivery checklist (15 items)
- Alignment scoring (1-10 scale with criteria)
- Minimum 7/10 threshold
- Automatic flagging for human review if <7

### Problem 5: Curated Foundations Had No Process

**Before:** When algorithm fails, manual research happens ad-hoc.

**After:** Explicit curated foundation process:
- When to use (algorithm doesn't match needs)
- Where to research (Candid, state associations, similar orgs' 990s)
- How to merge with algorithm output
- Generate snapshots for curated foundations

## Phase-by-Phase Summary

### Phase 0: Pre-Flight Checks

**New:** Verify environment and client data before starting.

### Phase 1: Client Understanding

**Goal:** Complete understanding before ANY algorithm runs.

**Key additions:**
- Customer type identification with signals and examples
- Multi-priority handling with minimum coverage rule
- Geographic specificity table (what client says vs. what algorithm does)
- Client Brief template

### Phase 2: Foundation Discovery

**Goal:** Best-fit foundations via algorithm AND manual curation.

**Key additions:**
- Decision tree for algorithm vs. manual research
- Quality check gate after top 20 review
- Curated foundation research process
- Merge process for scored + curated

### Phase 3: Enrichment & Research

**Goal:** Intelligence on each foundation for viability and positioning.

**Key additions:**
- Priority tiers (Tier 1: full research, Tier 2: basic, Tier 3: only if needed)
- Essential vs. nice-to-have fields
- Red flag identification
- Viability tiers (READY, CONDITIONAL, WATCH, SKIP)
- Minimum viable count (5)

### Phase 4: Report Assembly

**Goal:** Client-ready report with actionable opportunities.

**Key additions:**
- Selection criteria (viability, alignment, priority coverage, diversity)
- Narrative requirements (Why This Fits, Comparable Grant, Positioning, Next Steps)
- Customer-type-specific report formats

### Phase 5: Quality Checkpoints

**Goal:** Systematic quality assurance before delivery.

**Key additions:**
- 15-item pre-delivery checklist
- Alignment scoring (1-10 scale with criteria)
- Minimum 7/10 threshold
- Automatic human review flagging
- Post-delivery tracking
