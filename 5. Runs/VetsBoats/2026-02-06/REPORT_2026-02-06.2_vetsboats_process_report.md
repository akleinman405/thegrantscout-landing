## Lessons Learned

### What Worked Well

1. **Parallel research agents** — Running 3 research streams simultaneously (SF Challenge, 1687, connection paths) saved significant time during the pre-report gap-filling phase.

2. **Sail to Prevail as a bridge** — Discovering that both Barry and Van Beuren fund Sail to Prevail created the strongest insight of the report: one relationship unlocks two cultivation targets. This kind of cross-foundation intelligence is what differentiates TheGrantScout.

3. **Honest assessment of constraints** — Being upfront about VetsBoats' PF status and the narrow viable pool (rather than padding with weak matches) produced a more credible and useful report.

4. **DB-first enrichment before web research** — Pre-screening 8 of 17 foundations via database queries (geographic restrictions, PF exclusions) before doing any web research saved hours of wasted effort.

### What Could Improve

1. **PF clients need a different scoring pipeline** — The standard model doesn't account for PF-to-PF eligibility. A future enhancement should add a "PF grants count" feature to the scorer so PF clients get foundations that have actually funded other PFs ranked higher.

2. **Foundation enrichment caching** — The enrichment data gathered for these 17 foundations should be stored in the database (foundation_enrichment table) so future reports for VetsBoats or similar clients don't need to re-research. This was not done in this run.

3. **Connection path analysis could be automated** — The Matt Goettelman → Ron Levin → Kovler connection was found manually through web research. A future enhancement could cross-reference client advisory board members against foundation officer records in the officers table.

4. **Report assembly took too long** — Writing the full 5-foundation report from scratch required multiple sessions. A template with pre-filled sections and merge fields would reduce this to a single session.

### Process Improvements for Next Client

- Add PF-to-PF grant count as a scoring feature for PF clients
- Store enrichment results in DB after each run
- Build a connection path finder that queries the officers table
- Create a report assembly script that generates the markdown from structured data (07_report_data.json → 08_report.md path in the pipeline)
