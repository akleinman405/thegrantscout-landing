## Issues Found & Fixed

### Category 1: Bold Labels Before Tables (4 instances)
`**Bold Label:**` immediately followed by a markdown table with no blank line caused the table to render as inline text with pipe characters visible.

**Affected sections:**
- Recommended LOI Package (Bill Simpson)
- LOI Requirements (Herzstein)
- LOI Requirements (Kovler)
- Recommended Approach (Van Beuren)

**Fix:** Added blank line between each bold label and its table.

### Category 2: Bold Labels Before Lists (6 instances)
Same pattern as above but with bullet/numbered lists instead of tables.

**Affected sections:**
- Recommended actions (PF Advisory)
- Note: Cultivation criteria (Foundation Types)
- After You Submit (Bill Simpson, Herzstein, Kovler)
- Shared Grantees (Barry)

**Fix:** Added blank line between each bold label and its list.

### Category 3: Consecutive Bold Metadata Lines (~15 instances)
Multiple `**Bold:**` lines in a row (e.g., Foundation Website / Phone / Address / EIN) rendered as one merged paragraph.

**Affected sections:**
- Report header metadata (Date, Period, Profiled, Funding)
- All 5 foundation profile headers (Type / Time / Award)
- Bill Simpson contacts (Website / Key Officers)
- Herzstein contacts (Website / Portal / Phone)
- Barry contacts (Website / Professional Contact)
- Van Beuren contacts (Website / Phone / Address / EIN / Giving Focus)
- Van Beuren action summary (Total Active Time / July Deadline)
- Report footer (Prepared by / Date / Next Report)
- Data Sources section (4 consecutive lines)

**Fix:** Added blank line between each consecutive bold metadata line.

### Category 4: Code Blocks Collapsing (2 instances)
Triple-backtick code blocks rendered as single wrapped paragraphs in PDF, losing all line breaks.

**Affected sections:**
- Timeline flowchart (MONTH 1 → MONTHS 6-9)
- Dependencies flowchart (5 foundation process flows)

**Fix:** Converted timeline to bullet list, dependencies to table.

### Category 5: Blank Page with Heading (1 instance)
Page breaks both before AND after "Foundation Profiles" heading left the heading isolated on a blank page.

**Fix:** Removed the page break after the heading so content flows immediately.

### Category 6: Em Dashes (71 instances)
` --- ` em dashes throughout the report created an AI-generated tone.

**Fix:** Python script replaced all 71 with context-appropriate punctuation:
- Commas for light pauses
- Periods for independent clauses
- Colons for explanations
- Preserved standalone `---` in table cells (N/A markers)

### Category 7: Non-Clickable URLs (5 instances)
Foundation website references were plain text, not clickable in PDF.

**Fix:** Added markdown link syntax for:
- billsimpsonfoundation.org
- herzsteinfoundation.org (+ application portal)
- barryfamilyfoundation.org
- prospectcap.com
- vbcfoundation.org (already had link)

Also added `text-decoration: underline` to CSS for visual link indication.

## Notes

### Root Cause
All formatting issues stemmed from one markdown behavior: the Python `markdown` library (and standard CommonMark) requires blank lines to separate block elements. When a paragraph (like a `**bold:**` label) is followed by a list or table without a blank line, the parser treats it as continuation of the paragraph rather than a new block element.

### Recommendation: Pre-Delivery Formatting Lint
Consider adding a Python script that automatically checks for and fixes these patterns before PDF generation:
- Bold labels followed by `|` table rows without blank lines
- Bold labels followed by `- ` or `1. ` list items without blank lines
- Consecutive bold metadata lines without blank lines
- Code blocks (``` markers)
- Em dashes in prose

This could be integrated as step 8.5 in the pipeline, between `08_render_report.py` and PDF/DOCX generation.
