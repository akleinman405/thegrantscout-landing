## Challenges Encountered

### 1. State Field Truncation
**Issue:** `dim_clients.state` stored full state names ("California") but `calc_client_foundation_scores.foundation_state` is VARCHAR(2).

**Resolution:**
- Updated dim_clients to use 2-letter abbreviations
- Added `LEFT(state, 2)` in queries for foundation state

**Pipeline Improvement:** Enforce 2-letter state codes at data entry.

### 2. Recipient State Data Quality
**Issue:** `fact_grants.recipient_state` contains invalid values (e.g., "?le de France", numeric codes, international locations).

**Resolution:** Added `LEFT(recipient_state, 2)` truncation in grant population query.

**Pipeline Improvement:** Add data validation for recipient_state field.

### 3. Too Many Siblings for Generic Missions
**Issue:** SNS mission ("senior citizens and people with disabilities with information, guidance and assistance") matched 12,402+ nonprofits.

**Resolution:** Added `LIMIT 500` to sibling query, ordering by similarity DESC to keep best matches.

**Pipeline Improvement:** Consider mission specificity scoring or automatic threshold adjustment.

### 4. PSMF Low Target Grant Count
**Issue:** PSMF only found 1 target grant despite 298 sibling grants.

**Root Cause:** Keywords ("patient safety", "healthcare education", "clinical education") are highly specific. Most grant purposes use broader terms.

**Recommendation:** For healthcare clients, add broader keywords like "health education", "medical training", "clinical program", "healthcare improvement".

### 5. Previous Data Inconsistency (PSMF)
**Issue:** PSMF had prior data with different `purpose_quality` schema (high/medium/low/poor vs VALID/NULL/EMPTY/SHORT).

**Resolution:** Cleared and re-ran complete pipeline from scratch.

**Pipeline Improvement:** Version the calc table schema.
