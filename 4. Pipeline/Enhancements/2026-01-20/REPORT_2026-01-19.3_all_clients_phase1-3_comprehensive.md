## Challenges & Resolutions

### Challenge 1: State Field Data Type Mismatch

**Issue:** `dim_clients.state` stored full state names ("California", "North Carolina") but `calc_client_foundation_scores.foundation_state` is VARCHAR(2).

**Error:** `StringDataRightTruncation: value too long for type character varying(2)`

**Resolution:**
1. Updated `dim_clients` to use 2-letter state abbreviations
2. Added `LEFT(df.state, 2)` in aggregation queries

**Impact:** All 7 clients affected initially.

### Challenge 2: Invalid Recipient State Values

**Issue:** `fact_grants.recipient_state` contains invalid values from international grants:
- "?le de France" (France)
- "'AMMAN" (Jordan)
- Numeric ZIP codes ("00185", "0021")

**Resolution:** Added `LEFT(fg.recipient_state, 2)` truncation in grant population query.

**Impact:** Affected SNS and Ka Ulukoa (larger sibling pools).

### Challenge 3: Too Many Siblings for Generic Missions

**Issue:** SNS mission ("senior citizens and people with disabilities with information, guidance and assistance") matched 12,402 nonprofits - too broad for efficient processing.

**Root Cause:** Mission text is generic, matching many social service organizations.

**Resolution:** Added `LIMIT 500` to sibling query, ordering by similarity DESC to retain best matches.

**Decision Made:** 500 siblings is sufficient for discovery while maintaining processing efficiency.

### Challenge 4: PSMF Low Target Grant Count

**Issue:** PSMF found only 1 target grant despite 298 sibling grants.

**Root Cause:** Keywords are too specific:
- "patient safety"
- "healthcare education"
- "clinical education"
- "medical education"
- "fellowship program"

Most grant purposes use broader terms like "health education", "medical training", or "healthcare improvement".

**Resolution:** Documented for follow-up. Recommend broader keywords.

**Learning:** Healthcare sector needs broader keyword coverage due to varied terminology.

### Challenge 5: Previous Data Schema Inconsistency (PSMF)

**Issue:** PSMF had prior data with different `purpose_quality` values (high/medium/low/poor) vs expected (VALID/NULL/EMPTY/SHORT).

**Resolution:** Cleared existing data and re-ran complete pipeline from scratch.

**Learning:** Need schema versioning for calc tables.

### Challenge 6: Python f-string Backslash Limitation

**Issue:** Python f-strings don't allow backslashes in expressions.

**Error:** `SyntaxError: f-string expression part cannot include a backslash`

**Resolution:** Created helper function `make_keyword_pattern()` to build regex patterns outside f-string.

## Key Learnings

### 1. Mission Specificity Matters
- Generic missions (SNS) find many siblings but dilute signal
- Specific missions (PSMF) find fewer siblings but may miss relevant matches
- **Sweet spot:** 50-200 siblings with similarity >0.50

### 2. Keyword Selection is Critical
- Too specific = low target grant count (PSMF)
- Too broad = noisy matches
- **Best practice:** Include both specific and broader synonyms (8-15 keywords)

### 3. Budget Range Affects Results
- Larger budgets (RHF at $42M) = stronger foundation matches
- Smaller budgets (PSMF at $365K) = smaller foundation pool
- Budget 0.2x-5.0x ratio is appropriate

### 4. Affiliate Networks Are Gold
- Horizons affiliates funded by same foundations = direct signal
- Friendship Circle funded directly = strongest evidence
- **Opportunity:** Identify affiliate/network relationships for all clients

### 5. Data Quality Requires Ongoing Attention
- State codes, recipient states need validation
- Purpose text quality varies significantly
- Schema consistency needed for calc tables
