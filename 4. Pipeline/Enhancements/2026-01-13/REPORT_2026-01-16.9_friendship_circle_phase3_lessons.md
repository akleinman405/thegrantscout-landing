## Executive Summary

We executed Phase 3 (Foundation Discovery) for Friendship Circle San Diego, a Jewish nonprofit providing services to individuals with special needs in San Diego County. The process revealed **a critical flaw in the current methodology**: the "gold standard" foundations identified through sibling analysis are almost entirely **invite-only** and cannot be approached through applications.

### Key Metrics

| Metric | Value |
|--------|-------|
| Siblings Found | 102 |
| Sibling Grants | 1,460 |
| Unique Foundations | 554 |
| Target Grants (semantic + keyword match) | 61 |
| Foundations with Target Grants | 38 |
| **Gold Standard Foundations** | **9** |
| Gold Standard that Accept Applications | **2** |
| **Elimination Rate** | **78%** |

### The Core Problem

**78% of gold standard foundations are invite-only** per their IRS 990-PF filings. The current process spends significant effort identifying "best fit" foundations through sibling analysis, only to discover at the enrichment phase that most cannot be approached. This is backwards.

## Critical Problem Identified

### The Current Process Is Backwards

**Current Flow:**
```
Phase 3.1-3.3: Score foundations based on sibling matches
    ↓ (Significant effort)
Phase 3.4: Check if they accept applications
    ↓ (78% eliminated)
Phase 3.5: Select from remaining
    ↓ (Very limited pool)
```

**Problems:**
1. We spend significant effort scoring foundations that can't be approached
2. "Gold standard" metric is misleading - it measures FIT but not ACCESSIBILITY
3. The best-fit foundations often CAN'T be applied to
4. By Phase 3.4, we've invested heavily in foundations we must discard

### Proposed Flow

```
Phase 3.1: Find siblings (unchanged)
    ↓
Phase 3.2 (NEW): Pre-filter foundations by accessibility
    - IRS preselected indicator = FALSE
    - Has grant history in client's state
    - Minimum assets threshold
    ↓
Phase 3.3: Populate sibling grants (only from accessible foundations)
    ↓
Phase 3.4: Score accessible foundations
    ↓
Phase 3.5: Deep enrichment research on top prospects
    ↓
Phase 3.6: Final selection
```

## Process Observations

### What Worked Well

1. **Broader semantic framing** - Iteration on target purpose dramatically improved results
2. **AND logic for target grants** - Requiring both semantic AND keyword match increased precision
3. **Gold standard concept** - Target + first-time + geography IS a good signal, just needs accessibility filter first
4. **Session state documentation** - Enabled seamless continuation after conversation clear

### What Needs Improvement

1. **Accessibility check comes too late** - Should be first filter, not last
2. **No distinction between foundation types** - Private foundations vs. community foundations vs. corporate foundations have different application processes
3. **Geographic filtering is basic** - Needs to consider headquarters state vs. actual funding geography
4. **Website research is manual and slow** - Consider building a foundation enrichment cache
5. **IRS indicator is necessary but not sufficient** - Some "open" foundations don't accept unsolicited requests

## Recommended Process Changes

### Change 1: Add Accessibility Pre-Filter (New Step 3.1.5)

Before populating sibling grants, filter foundations to those likely accessible:

```sql
-- Create accessible foundations filter
CREATE TABLE calc_accessible_foundations AS
SELECT ein, name, state, assets
FROM f990_2025.dim_foundations
WHERE accepts_applications = TRUE  -- based on IRS indicator
  AND grants_to_orgs = TRUE
  AND assets >= 100000  -- meaningful capacity
  AND last_return_year >= 2022  -- still active
```

### Change 2: Only Score Accessible Foundations

Modify Step 3.2.1 to only pull grants from accessible foundations:

```sql
-- Current: All grants to siblings
INSERT INTO calc_client_sibling_grants ...
WHERE sibling_ein IN (SELECT ein FROM calc_client_siblings)

-- Proposed: Only grants from accessible foundations
INSERT INTO calc_client_sibling_grants ...
WHERE sibling_ein IN (SELECT ein FROM calc_client_siblings)
  AND foundation_ein IN (SELECT ein FROM calc_accessible_foundations)
```

### Change 3: Create "Gold Standard Accessible" Metric

Replace "Gold Standard" with two metrics:
- **Fit Score:** Target grants × first-time × geography (current gold standard)
- **Prospect Score:** Fit Score × accessibility indicator

### Change 4: Add Foundation Type Classification

Add foundation type to help prioritize research:
- **Private Family Foundation:** May be harder to approach despite IRS indicator
- **Private Operating Foundation:** Different rules
- **Corporate Foundation:** Often has formal application process
- **Community Foundation:** Usually has structured programs

### Change 5: Iterative Semantic Framing

Make Step 3.2.2 iterative:
1. Run with specific target purpose
2. If < 20 grants match, broaden framing
3. Document both results
4. Use broader framing for scoring, note specificity for positioning

## Data Quality Issues

### Issue 1: IRS Indicator Not Always Reliable

The Gilbert Foundation shows `only_contri_to_preselected_ind = FALSE` but their website says they don't accept unsolicited requests.

**Recommendation:** Add a `verified_accepts_applications` column populated through website research.

### Issue 2: Geographic Focus Not Captured

A Colorado foundation funding California organizations is different from a California foundation funding California. We don't track:
- Foundation headquarters state (we have this)
- Actual funding geography pattern (we could calculate)
- Stated geographic restrictions (requires research)

**Recommendation:** Add `primary_funding_states` array to dim_foundations.

### Issue 3: Community Foundations Not in Database

Jewish Community Foundation of San Diego is a major funder not captured in our private foundation (990-PF) data. Community foundations file 990s, not 990-PFs.

**Recommendation:** Consider adding community foundation data source.

## Appendix: Database Queries

### Query: Find Open Foundations with Target Grants

```sql
WITH latest_returns AS (
    SELECT DISTINCT ON (ein)
        ein, only_contri_to_preselected_ind
    FROM f990_2025.pf_returns
    ORDER BY ein, tax_year DESC
)
SELECT
    cfs.foundation_ein,
    cfs.foundation_name,
    cfs.foundation_state,
    cfs.gold_standard_grants,
    cfs.target_grants_to_siblings,
    lr.only_contri_to_preselected_ind as invite_only
FROM f990_2025.calc_client_foundation_scores cfs
LEFT JOIN latest_returns lr ON lr.ein = cfs.foundation_ein
WHERE cfs.client_ein = '203472700'
  AND (lr.only_contri_to_preselected_ind = FALSE
       OR lr.only_contri_to_preselected_ind IS NULL)
  AND cfs.target_grants_to_siblings > 0
ORDER BY cfs.target_grants_to_siblings DESC;
```

### Query: Check IRS Preselected Status

```sql
SELECT
    ein,
    business_name,
    state,
    total_assets_eoy_amt,
    only_contri_to_preselected_ind as invite_only,
    website_url
FROM f990_2025.pf_returns
WHERE ein = '753011479'  -- Foundation EIN
ORDER BY tax_year DESC
LIMIT 1;
```

### Query: View Actual Grants from a Foundation

```sql
SELECT
    df.name as foundation,
    dr.name as recipient,
    g.recipient_state,
    g.amount,
    g.purpose_text,
    g.tax_year,
    g.is_target_grant,
    g.semantic_similarity
FROM f990_2025.calc_client_sibling_grants g
JOIN f990_2025.dim_foundations df ON df.ein = g.foundation_ein
JOIN f990_2025.dim_recipients dr ON dr.ein = g.sibling_ein
WHERE g.client_ein = '203472700'
  AND g.foundation_ein = '203198081'  -- Ekstrom Foundation
ORDER BY g.tax_year DESC;
```
