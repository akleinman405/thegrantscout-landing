## BMF Table Import

### What Is the BMF?

The IRS Business Master File contains every organization that has ever received a tax-exempt determination letter. Unlike our XML filing data (which only covers orgs that file 990/990-PF returns), the BMF includes churches (290K, exempt from filing), very small orgs filing 990-N e-Postcards (146K), and all non-501(c)(3) exempt organizations.

### Why More Orgs Than Our XML Data?

Our XML filing data has 143K foundation EINs + 673K nonprofit EINs = ~816K. The BMF has 1.61M 501(c)(3)s. The ~800K gap consists of:

- **Churches (290K)** -- exempt from filing Form 990 entirely
- **990-N e-Postcard filers (146K)** -- gross receipts under $50K, no XML return
- **990-EZ filers not in our download** -- depending on which TEOS batches we pulled

## Notes

### Recommendations

1. **Add `bmf_status = '01'` filter to scoring pipeline** -- The 5,590 revoked foundations and 20K not-in-BMF foundations should be excluded from client reports.
2. **NTEE codes are not always accurate** -- Goodwill's N320 misclassification shows IRS NTEE codes should be treated as a signal, not gospel. Cross-reference with mission text for better sector matching.
3. **30% of BMF orgs lack NTEE codes** -- 583K organizations have NULL ntee_cd. This affects our `bmf_ntee_cd` enrichment on prospects2 tables too.
4. **Nature Conservancy missing NTEE** -- Even $9.5B organizations can have NULL NTEE. Don't assume large orgs always have complete data.
5. **BMF refreshes periodically** -- The IRS updates the BMF monthly. Consider re-downloading and reloading periodically to catch status changes (revocations, new orgs).

### Issues Encountered

- MCP postgres tool enters aborted transaction state after errors; used psql via Bash for all multi-query work.
- COPY command requires `SET search_path TO f990_2025` (schema-qualified table name doesn't work with psycopg2 `copy_from`).
