## Implementation Details

### 1. Fix Narrative Merge Bug (07_build_report_data.py)

**Problem:** Narratives generated in step 06 were silently failing to merge into opportunities in step 07. Reports rendered with empty `why_this_fits`, `positioning_strategy`, and `next_steps` fields.

**Root cause:** The merge code used `narrative_map.get(ein, {})` which returned an empty dict when narrative was missing, then silently extracted empty strings with `.get("why_this_fits", "")`.

**Fix:** Added explicit tracking and logging:
- Count how many narratives successfully merge
- Track which opportunities are missing narratives and why
- Print clear warnings showing missing EINs
- Log critical error if ALL narratives fail to merge

**Code location:** `scripts/07_build_report_data.py`, lines 282-398

**Key changes:**
```python
# Before: Silent failure
narr = narrative_map.get(ein, {})
opp["why_this_fits"] = narr.get("why_this_fits", "")

# After: Explicit tracking
merged_count = 0
missing_narratives = []

for opp in opportunities:
    narr = narrative_map.get(ein)
    if narr:
        # Merge and track
        has_content = bool(opp["why_this_fits"] or ...)
        if has_content:
            merged_count += 1
        else:
            missing_narratives.append(f"{name} - empty")
    else:
        missing_narratives.append(f"{name} - not found")

# Print summary and warnings
if merged_count == 0 and len(opportunities) > 0:
    print("CRITICAL: No narratives were merged!")
```

### 2. Expand Scored Pool to 500 (02_score_foundations_v6.1.py)

**Problem:** Default pool of 100 foundations was too narrow. When filtering by client-specific criteria (geography, grant type, sector), the pool shrunk too quickly.

**Fix:** Changed `--top-k` default from 50 to 500.

**Code location:** `scripts/02_score_foundations_v6.1.py`, line 690

**Before:**
```python
parser.add_argument('--top-k', type=int, default=50, ...)
```

**After:**
```python
parser.add_argument('--top-k', type=int, default=500,
    help='Number of foundations to return (default: 500 for more filtering options)')
```

**Behavior:** Users can still specify smaller pools with `--top-k 100` if desired.

### 3. Curated Foundation Snapshot Script (02b_add_foundation.py)

**Problem:** When manual research found good foundation matches not in the scored list, we had no way to generate their snapshot data. The RHF report initially had 0 opportunities because curated foundations had no snapshots.

**Solution:** New script that generates snapshots for any EIN(s), using the same SQL queries as `03_get_snapshots.py`.

**Code location:** `scripts/02b_add_foundation.py` (new file, 317 lines)

**Features:**
- Accept EINs via `--ein`, `--eins` (comma-separated), or `--file` (CSV or text)
- Generate full snapshots (annual giving, typical grant, geographic focus, etc.)
- Mark as `"source": "curated"` to distinguish from algorithmically scored
- Optional `--merge` flag to add to existing `03_snapshots.json`

**Usage:**
```bash
# Add single foundation by EIN
python scripts/02b_add_foundation.py --client "RHF" --ein "526037034"

# Add multiple foundations
python scripts/02b_add_foundation.py --client "RHF" --eins "526037034,581695425"

# Add from file
python scripts/02b_add_foundation.py --client "RHF" --file curated_foundations.csv

# Merge into existing snapshots
python scripts/02b_add_foundation.py --client "RHF" --ein "526037034" --merge
```
