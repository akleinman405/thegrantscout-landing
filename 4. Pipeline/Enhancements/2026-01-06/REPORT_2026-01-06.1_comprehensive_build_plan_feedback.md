## The Situation: What We Learned from 5 Client Reports

### Performance Summary

| Client | Alignment Score | Primary Gap |
|--------|-----------------|-------------|
| PSMF | 6/10 | No patient-safety-specific foundations |
| RHF | 7/10 | Missing Nevada and Utah coverage |
| SNS | 6/10 | Partnership/contract opportunities unaddressed |

| FCSD | 4/10 | **Geographic mismatch** (San Diego client got Aptos), capital project unaddressed |

**Average: 5.6/10**

### Root Causes Identified

1. **Geographic matching too coarse** - Algorithm uses state-level (`match_state_pct`). FCSD in San Diego got foundations in Aptos (400 miles away). Both are "California."

2. **Multi-priority not handled** - FCSD wanted bakery + building. Pipeline treats client as single entity, not per-priority.

3. **National orgs get local foundations** - HN operates nationally but got CT-only foundations (their HQ state).

4. **Client intake incomplete** - All 5 clients had empty `mission_statement`, empty `grant_size_target`.

5. **Narratives don't render** - Bug in step 07 causes narratives to not merge into opportunities.

## Agent Consensus: What's Missing from Build Plan

### Missing Item 1: Geographic Specificity

**Problem:** Scoring model uses `match_state_pct` (+1.69 coefficient). California has 58 counties. San Diego ≠ Aptos.

**Recommendation (all agents agree):**
1. Add city and county to client profiles (REQUIRED fields)
2. Calculate `match_city_pct` and `match_county_pct` features
3. Filter foundations by regional proximity before output

### Missing Item 2: Multi-Priority Handling

**Problem:** FCSD has two priorities (bakery + building). Pipeline only addressed bakery.

**Recommendation:**
- Run pipeline per-priority, not per-client
- FCSD-Bakery: Small grants, operating support, food focus
- FCSD-Building: Capital funders, larger grants
- Merge into single report with both covered

### Missing Item 3: Client Intake Validation

**Problem:** All 5 clients had empty `mission_statement`, `grant_size_target`.

**Recommendation:**
```
REQUIRED before pipeline runs:
[ ] Mission statement (min 50 words)
[ ] Grant size target (min/max range)
[ ] Geographic focus (city/county, not just state)
[ ] Top 3 specific priorities
[ ] Known funders to exclude
```

### Missing Item 4: Customer Type Identification

**Problem:** Different customers need different outputs (calendar managers vs. niche seekers vs. relationship builders).

**Recommendation:**
- Add customer type to questionnaire
- Create 2-3 report templates by type
- SOP now includes customer type identification (see Process Changes)
