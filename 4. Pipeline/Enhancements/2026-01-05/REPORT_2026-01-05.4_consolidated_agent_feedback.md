## Cross-Cutting Themes

### Theme 1: Data Exists But Doesn't Flow

| Data Type | Source | Destination | Current Status |
|-----------|--------|-------------|----------------|
| Funder snapshots | SQL queries | 03_snapshots.json | Working |
| Enrichment research | Manual web scraping | 05b_enrichment.json | Not persisted to DB |
| Narratives | 06_generate_narratives.py | 06_narratives.json | Not merged into opportunities |
| Template variables | 07_report_data.json | 08_render_report.py | Missing variables → fallback |

**Fix:** Create data flow diagram and ensure each step's output feeds the next step's input.

### Theme 2: Curated vs. Scored Foundation Tension

The LASSO V6.1 scoring model produces the top 100 foundations algorithmically, but:
- Manual research often identifies better matches (Weinberg, Woodruff, Kresge for RHF)
- These curated foundations aren't in the scored list → no snapshot data
- RHF got 0 opportunities initially because curated foundations had no snapshots

**Fix:** Create parallel track for curated foundations:
```
Scored Track: 01 → 02 → 03 → 04 → 05
Curated Track: Research → 03_curated → 04_curated → 05
Merge at: 05_combined → 06 → 07 → 08 → 09
```

### Theme 3: Geographic Matching is Too Coarse

| Client | Requested | Algorithm Matched | Gap |
|--------|-----------|-------------------|-----|
| FCSD | San Diego, CA | Aptos, CA (400 miles away) | Critical |
| RHF | DeKalb County, GA | Atlanta foundations | Good |
| RHF | Clark County, NV | None | Critical |
| HN | National | Connecticut only | Critical |

**Fix:** Add sub-state geography:
1. Client profiles need city/county, not just state
2. Foundation profiles need city/county from 990 address
3. Scoring model needs `match_city` and `match_county` features

### Theme 4: Multi-Priority Clients Get Partial Coverage

| Client | Priority 1 | Priority 2 | Addressed |
|--------|------------|------------|-----------|
| FCSD | Bakery expansion ($100K) | Community center (capital) | P1 only |

**Fix:** Pipeline should generate opportunities PER PRIORITY, not per client.

### 4. Log Template Variable Failures

Add to `08_render_report.py`:
```python
def render_with_logging(template, ctx):
    import re
    variables = re.findall(r'\{(\w+)\}', template)
    for var in variables:
        if var not in ctx or ctx.get(var) in [None, '', []]:
            print(f"WARNING: Missing template variable: {var}")
    return template.format(**{k: ctx.get(k, f"[MISSING: {k}]") for k in variables})
```
