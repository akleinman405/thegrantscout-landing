### 2. LEAKAGE: `f_funded_new_recently` Uses Future Data

**Problem:**
```sql
-- Current (WRONG)
CASE WHEN EXISTS (
    SELECT 1 FROM stg_first_time_grants
    WHERE tax_year >= 2023  -- Uses same year as prediction target!
) THEN 1 ELSE 0 END
```

**Fix:**
Modify `SQL_2025-01-02_v6_training_data.sql` line 218:

```sql
-- Fixed version (lag by 1-2 years)
CASE WHEN EXISTS (
    SELECT 1 FROM stg_first_time_grants ftg
    WHERE ftg.foundation_ein = tp.foundation_ein
      AND ftg.tax_year BETWEEN (tp.tax_year - 2) AND (tp.tax_year - 1)
) THEN 1 ELSE 0 END as f_funded_new_recently
```

**Or simpler:** Drop the feature entirely (likely redundant with `f_first_time_rate`)
