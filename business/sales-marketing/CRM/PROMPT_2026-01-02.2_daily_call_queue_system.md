# PROMPT_2026-01-02.2 - Daily Call Queue System

**Date:** 2026-01-02
**Type:** Feature Request (verbal instruction)

## Request Summary

User wants a daily call workflow system for the CRM with these capabilities:

### Core Features

1. **Daily Call Assignment**
   - Assign prospects to call on specific days
   - Open app, see day's call list, work through it
   - Check off each call as completed

2. **Called Section**
   - Once checked off, prospect moves to "called" section
   - Clear separation between to-do and done

3. **Follow-up Scheduling**
   - Mark a follow-up date during/after call
   - Prospect auto-appears on that day's queue
   - Recurring follow-up support

4. **Queue Page Date Filtering**
   - Toggle which dates to view
   - See today's calls, tomorrow's, this week's, etc.

5. **Company Description Display**
   - User reports not seeing descriptions
   - Need to verify display in queue and prospect views

### AI Integration (Future)

6. **Schedule-Based Planning**
   - Upload weekly schedule with commitments
   - AI analyzes available time for calls
   - Suggests number of calls per day
   - Auto-assigns high-priority prospects to queue

## Expected Deliverables

- Database schema changes (scheduled_call_date, follow_up_date fields)
- Updated Queue view with date picker/filter
- Check-off workflow for daily calls
- Description display fix
- Plan for AI schedule integration

## Technical Context

- PWA with Supabase backend
- Current tables: prospects, calls, emails, tasks, source_lists
- Current views: v_call_queue, v_dashboard_stats, v_todays_followups
- Frontend: Vanilla JS, deployed on Cloudflare Pages
