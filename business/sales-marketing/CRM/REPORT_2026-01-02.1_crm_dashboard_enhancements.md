## Addendum: Contacted Prospects Fix

### Issue
User reported that contacted prospects weren't showing in the prospects table.

### Investigation
- Total prospects in database: **80,378**
- Prospects with calls (call_count > 0): **76**
- The API was only loading 1,000 records sorted by segment/name
- The 76 called prospects were scattered among 80k, so many were cut off

### Root Cause
The prospects view loaded 1,000 records sorted alphabetically, causing called prospects to be randomly distributed and often not in the first 1,000 loaded.

### Fix Applied
1. **Changed API sort order** to `call_count.desc.nullslast,last_call_date.desc.nullslast,org_name`
   - Called prospects now appear first in results
2. **Added "Scheduled Only" filter** that loads only prospects with pending tasks (240 scheduled)
3. **Server-side filtering** for "Called Only" and "Scheduled Only" options
4. **Increased limit** from 1,000 to 2,000 records

|------|--------|
| `frontend/js/api.js` | New sort order, `getProspectsWithScheduledTasks()` method |
| `frontend/js/views/prospects.js` | "Scheduled Only" filter option, server-side reload |
