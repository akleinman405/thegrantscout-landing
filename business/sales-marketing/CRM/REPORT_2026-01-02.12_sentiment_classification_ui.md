## Implementation Details

### 1. API Method (api.js)

Added new method to update conversation sentiment:

```javascript
async updateConversationSentiment(conversationId, sentiment) {
    return this.request(`email_conversations?id=eq.${conversationId}`, {
        method: 'PATCH',
        body: JSON.stringify({ reply_sentiment: sentiment })
    });
}
```

### 2. UI Changes (prospect.js)

**For inbound emails only:**
- Dropdown with 7 sentiment options (Unclassified, Interested, Not Interested, Question, Meeting, Forwarded, Out of Office)
- Save button next to dropdown
- Green checkmark (✓) shown when email is already classified
- Pre-selects current sentiment value from database

**Styling:**
- Uses existing `form-select` and `btn-small` classes
- Compact inline layout (dropdown + button + checkmark)
- 160px dropdown width, small font size (0.875rem)

### 3. Save Handler

Added `saveSentiment(conversationId)` method:
- Reads value from dropdown
- Calls API to update database
- Updates local data model
- Shows toast "Sentiment saved" on success
- Re-renders view to show checkmark
