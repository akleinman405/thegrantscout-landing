**Date:** 2026-01-03

**Status:** Proposal

## Executive Summary

This document proposes a **Unified ICP Scoring Model** that predicts success across all outreach channels (phone, email) and all segments (nonprofits, foundation management companies). The model combines three scoring components:

| Component | Points | Purpose |
|-----------|--------|---------|
| **Base Score** | 0-40 | Organization characteristics & fit |
| **Channel Score** | 0-30 | Likelihood to respond by channel |
| **Readiness Score** | 0-30 | Likelihood to convert if reached |
| **Total** | 0-100 | Unified ICP Score |

## Component 1: Base Score (0-40 points)

Base score measures **organizational fit** - how likely is this organization to be a good customer?

### 1.1 Sector Alignment (0-12 points)

| Sector | Points | Rationale |
|--------|--------|-----------|
| **E** (Education) | 12 | 7.2x lift in call data |
| **P** (Human Services) | 12 | 7.2x lift in call data |
| **B** (Arts & Culture) | 8 | Often grant-dependent |
| **N** (Recreation/Sports) | 8 | Often grant-dependent |
| **Q** (International) | 6 | Strong grant need |
| **A** (Arts) | 6 | Often grant-dependent |
| **C** (Environment) | 5 | Standard fit |
| **D** (Animal-related) | 5 | Standard fit |
| **F** (Mental Health) | 5 | Standard fit |
| All others | 3 | Baseline |
| **Z** (Unknown) | 0 | Deprioritize |

**SQL:**
```sql
CASE sector
    WHEN 'E' THEN 12  -- Education
    WHEN 'P' THEN 12  -- Human Services
    WHEN 'B' THEN 8   -- Arts & Culture
    WHEN 'N' THEN 8   -- Recreation
    WHEN 'Q' THEN 6   -- International
    WHEN 'A' THEN 6   -- Arts
    WHEN 'C' THEN 5   -- Environment
    WHEN 'D' THEN 5   -- Animal
    WHEN 'F' THEN 5   -- Mental Health
    WHEN 'Z' THEN 0   -- Unknown
    ELSE 3
END as sector_score
```

### 1.2 Revenue Band (0-10 points)

Revenue $1M-$2M showed highest interest in call data.

| Revenue Band | Points | Rationale |
|--------------|--------|-----------|
| $1M-$2M | 10 | All 3 interested orgs in this band |
| $2M-$3.5M | 8 | Large enough for grant needs |
| $500K-$1M | 6 | Active grant seekers |
| $3.5M-$5M | 4 | May have dedicated staff already |
| >$5M | 2 | Too large, internal resources |
| <$500K | 2 | Too small, limited capacity |

**SQL:**
```sql
CASE revenue_band
    WHEN '$1M-$2M' THEN 10
    WHEN '$2M-$3.5M' THEN 8
    WHEN '$500K-$1M' THEN 6
    WHEN '$3.5M-$5M' THEN 4
    WHEN '>$5M' THEN 2
    ELSE 2
END as revenue_score
```

### 1.3 Staff Size (0-8 points)

Small orgs (1-10 employees) showed 6.67x interest lift.

| Staff Size | Points | Rationale |
|------------|--------|-----------|
| 1-10 employees | 8 | 6.67x lift for interest |
| 11-25 employees | 6 | Sweet spot |
| 26-50 employees | 4 | May have development staff |
| 51-100 employees | 2 | Likely has development team |
| 100+ employees | 1 | Internal resources |
| 0 employees | 4 | Volunteer-run, variable |

**SQL:**
```sql
CASE
    WHEN employee_count BETWEEN 1 AND 10 THEN 8
    WHEN employee_count BETWEEN 11 AND 25 THEN 6
    WHEN employee_count BETWEEN 26 AND 50 THEN 4
    WHEN employee_count BETWEEN 51 AND 100 THEN 2
    WHEN employee_count > 100 THEN 1
    ELSE 4  -- 0 or null employees
END as staff_score
```

### 1.4 Grant History (0-10 points)

Prior foundation grants indicate both fit AND responsiveness.

| Grant History | Points | Rationale |
|---------------|--------|-----------|
| 5+ unique funders | 10 | Actively pursuing grants |
| 2-4 unique funders | 8 | Growing grant portfolio |
| 1 unique funder | 6 | Single funder = vulnerability |
| Recent grant (2023+) | 4 | If no match but recent activity |
| No grants | 2 | New to foundations (lower answer rate) |

**SQL:**
```sql
CASE
    WHEN num_unique_funders >= 5 THEN 10
    WHEN num_unique_funders BETWEEN 2 AND 4 THEN 8
    WHEN num_unique_funders = 1 THEN 6
    WHEN most_recent_grant_year >= 2023 THEN 4
    ELSE 2
END as grant_history_score
```

## Component 2: Channel Score (0-30 points)

Channel score predicts **likelihood to respond** by outreach channel.

### 2.1 Phone Accessibility (0-15 points)

| Signal | Points | Rationale |
|--------|--------|-----------|
| NOT new to foundations | 5 | 2.4x more likely to answer |
| Has website (contact page) | 3 | More professional/responsive |
| Direct phone (vs. switchboard) | 3 | If contact_phone appears direct |
| Places found (Google Maps) | 2 | Verified business |
| Small staff (1-25) | 2 | Decision-maker answers |

**SQL:**
```sql
(CASE WHEN new_to_foundations = 0 THEN 5 ELSE 0 END) +
(CASE WHEN website IS NOT NULL AND website != '' THEN 3 ELSE 0 END) +
(CASE WHEN contact_phone LIKE '%ext%' OR LENGTH(REGEXP_REPLACE(contact_phone, '[^0-9]', '', 'g')) = 10 THEN 3 ELSE 1 END) +
(CASE WHEN places_found = true THEN 2 ELSE 0 END) +
(CASE WHEN employee_count BETWEEN 1 AND 25 THEN 2 ELSE 0 END)
as phone_score
```

### 2.2 Email Accessibility (0-15 points)

| Signal | Points | Rationale |
|--------|--------|-----------|
| Has email address | 5 | Required for email outreach |
| Has website with contact | 4 | Can find email if needed |
| Has concrete mission | 3 | Easier to personalize |
| Active online presence | 3 | Places found or verified website |

**Note:** Email data is limited. This component needs enrichment.

**SQL:**
```sql
(CASE WHEN contact_email IS NOT NULL AND contact_email != '' THEN 5 ELSE 0 END) +
(CASE WHEN website IS NOT NULL AND website != '' THEN 4 ELSE 0 END) +
(CASE WHEN has_concrete_mission = 1 THEN 3 ELSE 0 END) +
(CASE WHEN places_found = true OR (website IS NOT NULL AND website != '') THEN 3 ELSE 0 END)
as email_score
```

### 2.3 Combined Channel Score

For now, since email data is sparse, use phone score directly:

```sql
phone_score as channel_score  -- Until email enrichment complete
```

When email data is available:
```sql
GREATEST(phone_score, email_score) as channel_score  -- Best channel wins
```

## Component 3: Readiness Score (0-30 points)

Readiness score predicts **likelihood to convert if reached**.

### 3.1 Has Business Development Staff (0-12 points)

This is the CRITICAL finding: orgs WITH biz dev staff are 7x more likely to show interest.

| Has Biz Dev Staff | Points | Rationale |
|-------------------|--------|-----------|
| **yes** | 12 | 7x lift for interest |
| **uncertain** | 6 | Unclear |
| **no** | 2 | Easy to reach, hard to convert |

**SQL:**
```sql
CASE has_biz_dev_staff
    WHEN 'yes' THEN 12
    WHEN 'uncertain' THEN 6
    WHEN 'no' THEN 2
    ELSE 4
END as biz_dev_score
```

### 3.2 Recent Grant Activity (0-8 points)

Active grant-seeking indicates readiness.

| Activity | Points | Rationale |
|----------|--------|-----------|
| Grant in 2024 | 8 | Currently active |
| Grant in 2023 | 6 | Recently active |
| Grant in 2021-2022 | 4 | Medium recency |
| Grant pre-2021 | 2 | May be dormant |
| No grants | 2 | Unknown |

**SQL:**
```sql
CASE
    WHEN most_recent_grant_year >= 2024 THEN 8
    WHEN most_recent_grant_year = 2023 THEN 6
    WHEN most_recent_grant_year BETWEEN 2021 AND 2022 THEN 4
    WHEN most_recent_grant_year < 2021 AND most_recent_grant_year IS NOT NULL THEN 2
    ELSE 2
END as recent_activity_score
```

### 3.3 Concrete Mission (0-6 points)

Specific missions are easier to match to foundations.

| Mission Type | Points | Rationale |
|--------------|--------|-----------|
| Concrete mission (flagged) | 6 | Easy to match |
| Has mission but vague | 3 | Needs work |
| No mission | 0 | Hard to sell |

**SQL:**
```sql
CASE
    WHEN has_concrete_mission = 1 THEN 6
    WHEN mission_statement IS NOT NULL AND LENGTH(mission_statement) > 50 THEN 3
    ELSE 0
END as mission_score
```

### 3.4 Grant Dependency (0-4 points)

High grant dependency = high motivation.

| Dependency | Points | Rationale |
|------------|--------|-----------|
| >75% | 4 | Highly motivated |
| 50-75% | 3 | Motivated |
| 25-50% | 2 | Moderate |
| <25% | 1 | Low priority for grants |

**SQL:**
```sql
CASE
    WHEN grant_dependency_pct > 75 THEN 4
    WHEN grant_dependency_pct > 50 THEN 3
    WHEN grant_dependency_pct > 25 THEN 2
    ELSE 1
END as dependency_score
```

## Complete Scoring SQL

### For Nonprofits

```sql
WITH scores AS (
    SELECT
        ein,
        org_name,

        -- Component 1: Base Score (0-40)
        (
            -- 1.1 Sector (0-12)
            CASE sector
                WHEN 'E' THEN 12
                WHEN 'P' THEN 12
                WHEN 'B' THEN 8
                WHEN 'N' THEN 8
                WHEN 'Q' THEN 6
                WHEN 'A' THEN 6
                WHEN 'C' THEN 5
                WHEN 'D' THEN 5
                WHEN 'F' THEN 5
                WHEN 'Z' THEN 0
                ELSE 3
            END +
            -- 1.2 Revenue (0-10)
            CASE revenue_band
                WHEN '$1M-$2M' THEN 10
                WHEN '$2M-$3.5M' THEN 8
                WHEN '$500K-$1M' THEN 6
                WHEN '$3.5M-$5M' THEN 4
                ELSE 2
            END +
            -- 1.3 Staff (0-8)
            CASE
                WHEN employee_count BETWEEN 1 AND 10 THEN 8
                WHEN employee_count BETWEEN 11 AND 25 THEN 6
                WHEN employee_count BETWEEN 26 AND 50 THEN 4
                WHEN employee_count BETWEEN 51 AND 100 THEN 2
                WHEN employee_count > 100 THEN 1
                ELSE 4
            END +
            -- 1.4 Grant History (0-10)
            CASE
                WHEN num_unique_funders >= 5 THEN 10
                WHEN num_unique_funders BETWEEN 2 AND 4 THEN 8
                WHEN num_unique_funders = 1 THEN 6
                WHEN most_recent_grant_year >= 2023 THEN 4
                ELSE 2
            END
        ) as base_score,

        -- Component 2: Channel Score (0-30) - Phone focused
        (
            (CASE WHEN new_to_foundations = 0 THEN 5 ELSE 0 END) +
            (CASE WHEN website IS NOT NULL AND website != '' THEN 3 ELSE 0 END) +
            (CASE WHEN contact_phone IS NOT NULL AND contact_phone != '' THEN 3 ELSE 0 END) +
            (CASE WHEN places_found = true THEN 2 ELSE 0 END) +
            (CASE WHEN employee_count BETWEEN 1 AND 25 THEN 2 ELSE 0 END)
        ) * 2 as channel_score,  -- Scale to 0-30 range

        -- Component 3: Readiness Score (0-30)
        (
            -- 3.1 Biz Dev Staff (0-12)
            CASE has_biz_dev_staff
                WHEN 'yes' THEN 12
                WHEN 'uncertain' THEN 6
                WHEN 'no' THEN 2
                ELSE 4
            END +
            -- 3.2 Recent Activity (0-8)
            CASE
                WHEN most_recent_grant_year >= 2024 THEN 8
                WHEN most_recent_grant_year = 2023 THEN 6
                WHEN most_recent_grant_year BETWEEN 2021 AND 2022 THEN 4
                ELSE 2
            END +
            -- 3.3 Concrete Mission (0-6)
            CASE
                WHEN has_concrete_mission = 1 THEN 6
                WHEN mission_statement IS NOT NULL AND LENGTH(mission_statement) > 50 THEN 3
                ELSE 0
            END +
            -- 3.4 Grant Dependency (0-4)
            CASE
                WHEN grant_dependency_pct > 75 THEN 4
                WHEN grant_dependency_pct > 50 THEN 3
                WHEN grant_dependency_pct > 25 THEN 2
                ELSE 1
            END
        ) as readiness_score

    FROM f990_2025.prospects
    WHERE form_type != '990PF'
)
SELECT
    ein,
    org_name,
    base_score,
    channel_score,
    readiness_score,
    (base_score + channel_score + readiness_score) as unified_icp_score
FROM scores
ORDER BY unified_icp_score DESC;
```

## Key Design Decisions

### 1. Has Biz Dev Staff is Critical

The data shows orgs WITH biz dev staff are 7x more likely to show interest. This is the strongest signal for readiness.

**Implication:** The current `has_biz_dev_staff = 'no'` was treated as a positive (easier to reach), but it's actually a trap - they answer but don't convert. We flip this.

### 2. Sector E/P is the Best Fit

Education and Human Services sectors showed 7.2x lift for reaching decision makers.

**Implication:** These sectors get maximum sector score (12 points).

### 3. Revenue $1M-$2M is the Sweet Spot

All 3 interested organizations were in this revenue band.

**Implication:** $1M-$2M gets maximum revenue score (10 points).

### 4. "New to Foundations" is Negative for Phone

Orgs new to foundation funding are 2.4x LESS likely to answer the phone.

**Implication:** Phone accessibility score penalizes new-to-foundations.

### 5. Channel Score Enables Multi-Channel Strategy

Different orgs respond better to different channels. The model tracks both phone and email accessibility separately, allowing channel-optimized outreach.

## Appendix: Full Scoring Pseudocode

```python
def calculate_unified_icp(prospect):
    """Calculate unified ICP score for a nonprofit prospect"""

    # Component 1: Base Score (0-40)
    sector_score = {
        'E': 12, 'P': 12, 'B': 8, 'N': 8, 'Q': 6, 'A': 6,
        'C': 5, 'D': 5, 'F': 5, 'Z': 0
    }.get(prospect.sector, 3)

    revenue_score = {
        '$1M-$2M': 10, '$2M-$3.5M': 8, '$500K-$1M': 6, '$3.5M-$5M': 4
    }.get(prospect.revenue_band, 2)

    if 1 <= prospect.employee_count <= 10:
        staff_score = 8
    elif 11 <= prospect.employee_count <= 25:
        staff_score = 6
    elif 26 <= prospect.employee_count <= 50:
        staff_score = 4
    elif 51 <= prospect.employee_count <= 100:
        staff_score = 2
    elif prospect.employee_count > 100:
        staff_score = 1
    else:
        staff_score = 4

    if prospect.num_unique_funders >= 5:
        grant_score = 10
    elif prospect.num_unique_funders >= 2:
        grant_score = 8
    elif prospect.num_unique_funders == 1:
        grant_score = 6
    elif prospect.most_recent_grant_year >= 2023:
        grant_score = 4
    else:
        grant_score = 2

    base_score = sector_score + revenue_score + staff_score + grant_score

    # Component 2: Channel Score (0-30) - Phone focused
    phone_score = 0
    if not prospect.new_to_foundations:
        phone_score += 5
    if prospect.website:
        phone_score += 3
    if prospect.contact_phone:
        phone_score += 3
    if prospect.places_found:
        phone_score += 2
    if 1 <= prospect.employee_count <= 25:
        phone_score += 2

    channel_score = min(phone_score * 2, 30)  # Scale to 0-30

    # Component 3: Readiness Score (0-30)
    biz_dev_score = {'yes': 12, 'uncertain': 6, 'no': 2}.get(prospect.has_biz_dev_staff, 4)

    if prospect.most_recent_grant_year >= 2024:
        activity_score = 8
    elif prospect.most_recent_grant_year == 2023:
        activity_score = 6
    elif prospect.most_recent_grant_year >= 2021:
        activity_score = 4
    else:
        activity_score = 2

    if prospect.has_concrete_mission:
        mission_score = 6
    elif prospect.mission_statement and len(prospect.mission_statement) > 50:
        mission_score = 3
    else:
        mission_score = 0

    if prospect.grant_dependency_pct > 75:
        dependency_score = 4
    elif prospect.grant_dependency_pct > 50:
        dependency_score = 3
    elif prospect.grant_dependency_pct > 25:
        dependency_score = 2
    else:
        dependency_score = 1

    readiness_score = biz_dev_score + activity_score + mission_score + dependency_score

    # Unified Score
    unified_score = base_score + channel_score + readiness_score

    return {
        'base_score': base_score,
        'channel_score': channel_score,
        'readiness_score': readiness_score,
        'unified_icp_score': unified_score,
        'tier': get_tier(unified_score)
    }


def get_tier(score):
    if score >= 80:
        return 1
    elif score >= 65:
        return 2
    elif score >= 50:
        return 3
    elif score >= 35:
        return 4
    else:
        return 5
```
