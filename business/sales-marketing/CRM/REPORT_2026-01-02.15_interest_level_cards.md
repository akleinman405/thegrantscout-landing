### 1. `/frontend/js/views/prospects.js`

**Changes:**
- Added `formatInterest()` helper function to map interest values to icons and CSS classes
- Updated `renderCard()` to display interest badge in the last contact info section
- Interest badge appears after the outcome badge, before the date

**Helper function added:**
```javascript
function formatInterest(interest) {
    const icons = {
        'yes': { icon: '✓', class: 'interest-yes', label: 'Interested' },
        'no': { icon: '✗', class: 'interest-no', label: 'Not Interested' },
        'maybe': { icon: '?', class: 'interest-maybe', label: 'Maybe' },
        'uncertain': { icon: '~', class: 'interest-uncertain', label: 'Uncertain' }
    };
    return icons[interest] || null;
}
```

**In renderCard():**

## Visual Design

**Badge appearance:**
- **Yes (✓):** Green badge with checkmark icon
- **No (✗):** Red badge with X icon
- **Maybe (?):** Yellow/amber badge with question mark
- **Uncertain (~):** Gray badge with tilde

**Layout:**
```
[Talked] [✓] 3 days ago
[VM] [?] 1 week ago
[DM] [✗] Yesterday
```

The interest badge appears between the outcome badge and the timestamp, providing immediate visual feedback on prospect interest level.

---

## Data Source

The interest level comes from `v_prospects_with_last_call.last_call_interest`, which is populated from the most recent call's `interest` field.

**Interest values:**
- `yes` - Prospect expressed interest
- `no` - Prospect not interested
- `maybe` - Prospect unsure/needs more info
- `uncertain` - Interest level unclear
- `NULL` - No interest recorded (badge not shown)
