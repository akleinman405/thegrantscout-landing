## Context

The CRM has a `reply_sentiment` column in the `email_conversations` table that can hold values: interested, not_interested, question, meeting, forwarded, out_of_office, unclassified.

The Campaigns dashboard already displays sentiment counts, but there's NO UI to actually classify replies. All inbound emails have NULL sentiment.

## Task

Add a sentiment classification dropdown to each inbound email in the prospect detail view.

## Requirements

1. **Location:** `frontend/js/views/prospect.js` in the Email Conversations section

2. **For each inbound email (direction === 'inbound')**, show:
   - A dropdown/select with options: Unclassified, Interested, Not Interested, Question, Meeting, Out of Office
   - A "Save" button next to the dropdown
   - Visual indicator when already classified (checkmark or colored badge)

3. **API method needed** in `frontend/js/api.js`:
   ```javascript
   async updateConversationSentiment(conversationId, sentiment) {
       return this.request(`email_conversations?id=eq.${conversationId}`, {
           method: 'PATCH',
           body: JSON.stringify({ reply_sentiment: sentiment })
       });
   }
   ```

4. **Save handler:**
   - On save, call the API to update reply_sentiment
   - Show toast "Sentiment saved" on success
   - Update the UI to show the new classification

5. **Styling:**
   - Keep it compact (inline with email item)
   - Use existing form-select and btn-small classes
   - Green checkmark icon when classified
