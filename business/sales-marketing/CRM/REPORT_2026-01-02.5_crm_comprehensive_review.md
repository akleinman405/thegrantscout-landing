## Auto-Update Behavior

**Question:** Do tables automatically update when I do something like mark someone as contacted?

**Answer: YES, with limitations.**

### Automatic Updates (Working)

| Action | Auto-Updates |
|--------|--------------|
| Log a call | `prospects.status` -> 'contacted' (if was 'not_contacted') |
| Log a call | `prospects.last_contacted_at` -> now() |
| Log a call | `prospects.call_count` -> +1 |
| Create prospect | `prospects.timezone` -> derived from state |
| Update prospect | `prospects.updated_at` -> now() |

### Manual Updates Required

| Action | Requires Manual Update |
|--------|------------------------|
| Change status to 'interested' | Must click status chip in prospect detail |
| Change status to 'converted' | Must click status chip |
| Update prospect phone/email | Not possible in UI - requires database access |
| Edit call notes | Not possible - calls are immutable once logged |

## Editing Capabilities

**Question:** Is there a way to edit contact notes history for a nonprofit?

**Answer: NO.**

### What CAN Be Edited
- Prospect status (via chip buttons)
- Tasks (can be completed or deleted)
- Prospect notes (only via prospect detail, single field)

### What CANNOT Be Edited
- **Call records** - Once logged, call outcome, notes, and details are permanent
- **Email records** - Once logged, email details are permanent
- **Prospect details** - Phone, email, contact name, address, etc. are not editable in the UI
- **Historical notes** - No note history, just a single notes field

### Workaround (Current)
Must access Supabase dashboard directly or use SQL to edit:
```bash
# Access Supabase Studio
https://supabase.com/dashboard/project/qisbqmwtfzeiffgtlzpk

# Or use Python/SQL
UPDATE calls SET notes = 'Corrected note' WHERE id = 123;
```
