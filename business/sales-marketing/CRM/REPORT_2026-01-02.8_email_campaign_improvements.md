## New Behavior

### On Campaign Close (Ctrl+C or Natural End)

When either script closes, it now automatically:

1. **Marks script as stopped** in coordination.json
2. **Syncs to CRM** - Pushes today's emails to Supabase CRM
3. **Checks for bounces** - Uses Gmail API to find bounces/replies (last 7 days)
4. **Updates CSV files** - Marks bounces in sent_tracker.csv, replies in response_tracker.csv

### Periodic Sync During Day

Every 100 emails sent, the system automatically syncs to CRM. This ensures:
- CRM data stays current even during long sessions
- No data loss if script crashes unexpectedly

## Consolidation Proposal (Future)

Analyzed the current 3-script architecture:

| Script | Lines | Purpose |
|--------|-------|---------|
| `send_initial_outreach.py` | 1,100 | Initial emails |
| `send_followup.py` | 945 | Follow-up emails |
| `campaign_tracker.py` | 977 | Bounce/reply checking |

**Issues identified:**
- ~70% code duplication between initial and followup scripts
- Three separate scripts to manage
- Shared helper functions copied between files

**Recommended consolidation:** Create a single `campaign_manager.py` that:
```bash
python campaign_manager.py send              # Send both initial + followup
python campaign_manager.py send --type initial
python campaign_manager.py send --type followup
python campaign_manager.py check             # Check bounces/replies
python campaign_manager.py status            # Show stats
```

This would eliminate duplication and provide a single entry point. The current implementation with shutdown_hooks.py is a step toward this consolidation.
