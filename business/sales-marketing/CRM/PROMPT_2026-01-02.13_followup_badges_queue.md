# PROMPT_2026-01-02.13_followup_badges_queue.md

**Date:** 2026-01-02
**Type:** Implementation task for Claude Code CLI

## Context

When you log a call and set a follow-up date, a task is created. But when viewing the Queue, you can't see which prospects have pending follow-ups. Follow-ups are only visible in the Tasks view, causing users to miss them.

## Task

Add follow-up date badges to prospect cards in the Queue view's "Called Today" section.

## Requirements

1. **Location:** `frontend/js/views/queue.js` in the `renderCalledCard()` function

2. **For prospects with pending follow-ups**, show:
   - A badge next to the outcome showing "Follow-up: Jan 5" or "Follow-up: Tomorrow"
   - Color code: orange if due today, red if overdue, gray if future

3. **Data source:**
   - The `v_prospects_with_last_call` view already includes `follow_up_date` from the last call
   - Or query tasks table for pending call tasks

4. **Display format:**
   ```
   [Talked] [Follow-up: Tomorrow]  3 days ago
   ```

5. **Add quick reschedule button:**
   - Small "Reschedule" link next to follow-up badge
   - Opens ScheduleCallView modal with prospect pre-filled

## Files to Modify

| File | Changes |
|------|---------|
| frontend/js/views/queue.js | Add follow-up badge to renderCalledCard |
| frontend/css/style.css | Add badge-followup styles (optional) |

## Acceptance Criteria

- [ ] Called Today cards show follow-up date if set
- [ ] Badge is color-coded (overdue=red, today=orange, future=gray)
- [ ] Date is human-readable (Tomorrow, Jan 5, etc.)
- [ ] Reschedule link opens scheduling modal
