**Type:** Implementation task for Claude Code CLI

## Context

When logging a call, users can select an interest level (yes, no, maybe, uncertain). This is stored in the calls table as `interest`. However, this critical information is NOT shown on prospect cards in the list views.

The `v_prospects_with_last_call` view already includes `last_call_interest`, but the UI doesn't display it.

## Task

Show the last call interest level on prospect cards in the All Prospects view.

## Requirements

1. **Location:** `frontend/js/views/prospects.js` in the `renderCard()` function

2. **Display format:**
   - Show interest as an icon/badge next to the call outcome
   - Icons: ✓ (yes/green), ✗ (no/red), ? (maybe/yellow), ~ (uncertain/gray)
   - Or use text badges: "Interested", "Not Interested", "Maybe"

3. **Placement:**
   - In the last contact info section, after the outcome badge
   - Example: `[Talked] [Maybe] 3 days ago`

4. **Add helper function:**
   ```javascript
   function formatInterest(interest) {
       const icons = {
           'yes': { icon: '✓', class: 'interest-yes', label: 'Interested' },
           'no': { icon: '✗', class: 'interest-no', label: 'Not Interested' },
           'maybe': { icon: '?', class: 'interest-maybe', label: 'Maybe' },
           'uncertain': { icon: '~', class: 'interest-uncertain', label: 'Uncertain' }
       };
       return icons[interest] || null;
   }
   ```

5. **CSS (add to style.css):**
   ```css
   .interest-yes { color: var(--success); }
   .interest-no { color: var(--danger); }
   .interest-maybe { color: var(--warning); }
   .interest-uncertain { color: var(--gray-500); }
   ```
