## Automatic Behaviors (Triggers)

2. **Auto-update last_contacted_at:** Sets to current timestamp

### On Prospect Insert/Update

## Prioritization Logic

The call queue (`v_call_queue` view) orders prospects by:

```sql
ORDER BY
    tier NULLS LAST,              -- 1 = highest priority
    icp_score DESC NULLS LAST,    -- Higher = better fit
    created_at                     -- FIFO for equal priority
```

### Tier Assignments
| Tier | Description |
|------|-------------|
| 1 | Foundation management companies (top priority B2B) |
| 2 | Top 100 foundations by capacity grants |
| 3 | Foundations ranked 101-500 |
| 4 | Foundations ranked 500+ |
| 5-7 | Reserved for future use |
| NULL | Untiered prospects (lowest priority) |

### ICP Score Calculation
Based on foundation grant profiles:
- Capacity grants (weighted heavily)
- General support / unrestricted grants
- Phone responsiveness indicators (has named contact, email, phone)

## What Doesn't Work

### No Editing Capabilities
- Cannot edit prospect details (phone, email, contact name)
- Cannot edit call notes after logging
- Cannot edit email records
- Must use direct database access for corrections

### Limited Email Integration
- Manual email logging only
- No email campaign visibility
- No email conversation history
- No integration with email providers

### UI Friction Points
- Unscheduled section collapsed by default (extra click each time)
- No bulk actions (schedule multiple, update multiple)
- No inline editing
- Dashboard stats show incorrect totals (76K vs 1K actual)
