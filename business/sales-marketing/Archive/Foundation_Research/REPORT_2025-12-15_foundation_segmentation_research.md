## 4. Proposed Segments

### Segment 1: Large CB Foundations (SIZE + CB)

**Criteria:**
- Assets > $100M
- 50+ unique grantees
- At least 5 capacity-building grants
- Active in 2023-2024

**SQL Filter:**
```sql
WITH foundation_metrics AS (
    SELECT foundation_ein,
           COUNT(DISTINCT recipient_ein) as grantees,
           COUNT(*) FILTER (WHERE purpose_text ILIKE ANY(ARRAY[
               '%capacity building%', '%technical assistance%',
               '%organizational development%', '%general operating%',
               '%operating support%'
           ])) as cb_grants
    FROM f990_2025.fact_grants
    WHERE tax_year >= 2023
    GROUP BY foundation_ein
)
SELECT pr.ein, pr.business_name
FROM f990_2025.pf_returns pr
JOIN foundation_metrics fm ON pr.ein = fm.foundation_ein
WHERE pr.total_assets_eoy_amt > 100000000
  AND fm.grantees >= 50
  AND fm.cb_grants >= 5
  AND (pr.only_contri_to_preselected_ind = FALSE OR pr.only_contri_to_preselected_ind IS NULL)
```

**Estimated Count:** ~200-400 foundations
**Rationale:** Large foundations with CB history understand grantee support; high grantee volume = larger potential user base for TheGrantScout

## 5. Prioritization Recommendations

### Priority 1: Intermediary Funders

**Why first:**
- Already pay for grantee support services (evaluation, training, TA)
- Understand value proposition without explanation
- Decision-makers are accustomed to vendor relationships
- Natural fit for "help our grantees find more funding" pitch

**Estimated size:** 200-400 foundations
**Outreach approach:** Direct pitch to program staff; emphasize ROI for their grantee portfolio

---

### Priority 2: High-Volume Regional (CA or NY)

**Why second:**
- Geographic concentration enables efficient networking
- Large grantee pools = high value per sale
- Regional foundations often have accessible staff
- CA and NY have sophisticated nonprofit ecosystems

**Estimated size:** 500-800 foundations per state
**Outreach approach:** State-specific campaigns; leverage local nonprofit associations

---

### Priority 3: Large CB Foundations

**Why third:**
- Understand capacity building; may be receptive
- BUT: larger foundations are harder to reach
- Longer sales cycle expected
- Worth pursuing after learning from smaller foundations

**Estimated size:** 200-400 foundations
**Outreach approach:** Warm introductions; position as portfolio-wide efficiency tool

---

### Not Recommended for Initial Outreach

**Sector Specialists (Education):** Large segment but less likely to fund cross-sector tools
**Baseline/Random:** Use for testing only, not primary outreach

## 7. Phase 2 Inputs

### SQL Criteria for Each Priority Segment

**Segment A: Intermediary Funders**
```sql
WITH intermediary_funders AS (
    SELECT foundation_ein,
           COUNT(*) as intermediary_grants,
           COUNT(DISTINCT recipient_ein) as grantees
    FROM f990_2025.fact_grants
    WHERE purpose_text ILIKE ANY(ARRAY['%evaluation%', '%training%', '%technical assistance%', '%consulting%', '%assessment%'])
      AND tax_year >= 2022
    GROUP BY foundation_ein
    HAVING COUNT(*) >= 10 AND COUNT(DISTINCT recipient_ein) >= 50
)
SELECT pr.ein, pr.business_name, pr.state, pr.total_assets_eoy_amt,
       if.intermediary_grants, if.grantees
FROM intermediary_funders if
JOIN f990_2025.pf_returns pr ON if.foundation_ein = pr.ein
WHERE (pr.only_contri_to_preselected_ind = FALSE OR pr.only_contri_to_preselected_ind IS NULL)
ORDER BY if.grantees DESC;
```

**Segment B: CA High-Volume**
```sql
WITH ca_foundations AS (
    SELECT fg.foundation_ein, COUNT(DISTINCT fg.recipient_ein) as grantees
    FROM f990_2025.fact_grants fg
    JOIN f990_2025.pf_returns pr ON fg.foundation_ein = pr.ein
    WHERE pr.state = 'CA' AND fg.tax_year >= 2022
    GROUP BY fg.foundation_ein
    HAVING COUNT(DISTINCT fg.recipient_ein) >= 50
)
SELECT pr.ein, pr.business_name, pr.total_assets_eoy_amt, cf.grantees
FROM ca_foundations cf
JOIN f990_2025.pf_returns pr ON cf.foundation_ein = pr.ein
WHERE (pr.only_contri_to_preselected_ind = FALSE OR pr.only_contri_to_preselected_ind IS NULL)
ORDER BY cf.grantees DESC;
```

## 8. Lessons Learned

### What Worked
- CTE-based queries for complex metrics
- Using tax_year for recency filtering
- Breaking queries into smaller pieces to avoid timeouts

### What Was Harder
- Officer data lacks program staff (mostly trustees/presidents)
- Nevada registration anomaly required geographic analysis of grants, not foundation location
- Large ILIKE ANY queries slow on 8M+ rows

### Recommendations for Phase 2
1. **Pre-compute foundation metrics** into a summary table for faster querying
2. **Enrich officer data** with LinkedIn lookup for program staff
3. **Build state-specific views** for CA, NY, TX outreach campaigns
4. **Add website/email fields** from pf_returns for direct outreach
