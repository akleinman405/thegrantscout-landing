## Step 2: Create Prospects Table

Create table with this structure:

```sql
CREATE TABLE f990_2025.prospects (
    -- Primary Key
    ein VARCHAR(9) PRIMARY KEY,
    
    -- Base Info (from F990 import tables)
    org_name TEXT,
    state VARCHAR(2),
    city TEXT,
    zip VARCHAR(10),
    ntee_code VARCHAR(10),
    sector CHAR(1),  -- First letter of NTEE
    form_type VARCHAR(10),  -- 990, 990-EZ, etc.
    tax_year INT,
    
    -- Size Metrics
    total_revenue NUMERIC,
    total_expenses NUMERIC,
    total_assets NUMERIC,
    contributions_grants NUMERIC,
    program_service_revenue NUMERIC,
    grant_dependency_pct NUMERIC,
    employee_count INT,
    volunteer_count INT,
    
    -- Foundation Grants Received (from 990-PF grant data)
    num_foundation_grants INT DEFAULT 0,
    num_unique_funders INT DEFAULT 0,
    total_foundation_grant_amount NUMERIC DEFAULT 0,
    most_recent_grant_year INT,
    
    -- Calculated/Derived
    revenue_band VARCHAR(20),
    grant_dep_band VARCHAR(20),
    yoy_revenue_growth NUMERIC,
    
    -- Scoring
    icp_score INT DEFAULT 0,
    priority_tier INT,  -- 1, 2, or 3
    
    -- Enrichment (to be added later)
    website TEXT,
    mission_statement TEXT,
    contact_name TEXT,
    contact_title TEXT,
    contact_email TEXT,
    contact_phone TEXT,
    
    -- Personalization (for email generation)
    personalization_tier INT,
    personalization_hook TEXT,
    email_template_id TEXT,
    custom_sentence TEXT,
    email_draft TEXT,
    beta_client_connection TEXT,  -- "Same funder as Horizons", "Same sector as SNS"
    
    -- Outreach Tracking
    outreach_status VARCHAR(20) DEFAULT 'not_contacted',
    last_contact_date DATE,
    email_sent_date DATE,
    email_opened BOOLEAN,
    email_replied BOOLEAN,
    reply_sentiment VARCHAR(20),
    next_followup_date DATE,
    notes TEXT,
    
    -- Metadata
    created_at TIMESTAMP DEFAULT NOW(),
    updated_at TIMESTAMP DEFAULT NOW()
);

-- Indexes for common queries
CREATE INDEX idx_prospects_score ON f990_2025.prospects(icp_score DESC);
CREATE INDEX idx_prospects_tier ON f990_2025.prospects(priority_tier);
CREATE INDEX idx_prospects_status ON f990_2025.prospects(outreach_status);
CREATE INDEX idx_prospects_state ON f990_2025.prospects(state);
CREATE INDEX idx_prospects_sector ON f990_2025.prospects(sector);
```

## Step 3: Populate Base Data

Insert nonprofits matching these criteria:

**Inclusion Criteria:**
- Revenue: $500,000 - $5,000,000
- Grant dependency: > 25% (contributions_grants / total_revenue)
- Form type: 990 or 990-EZ (public charities)
- Tax year: Most recent available (2022-2024)
- Has some revenue (> 0)

**Exclusion Criteria (if detectable):**
- Universities/colleges (NTEE B40-B43)
- Hospitals (NTEE E20-E24)
- Churches (NTEE X)
- Federated giving (NTEE T)
- Government entities

**For each org, use most recent tax year filing.**

## Step 4: Add Foundation Grant History

Join with 990-PF grant data to count how many foundation grants each prospect has received:

```sql
-- Count foundation grants received by each prospect
UPDATE f990_2025.prospects p
SET 
    num_foundation_grants = g.grant_count,
    num_unique_funders = g.funder_count,
    total_foundation_grant_amount = g.total_amount,
    most_recent_grant_year = g.max_year
FROM (
    SELECT 
        recipient_ein,
        COUNT(*) as grant_count,
        COUNT(DISTINCT foundation_ein) as funder_count,
        SUM(amount) as total_amount,
        MAX(tax_year) as max_year
    FROM f990_2025.pf_grants  -- or whatever the grant table is called
    GROUP BY recipient_ein
) g
WHERE p.ein = g.recipient_ein;
```

## Step 5: Calculate Derived Fields

```sql
-- Revenue bands
UPDATE f990_2025.prospects
SET revenue_band = CASE
    WHEN total_revenue < 500000 THEN '<$500K'
    WHEN total_revenue < 1000000 THEN '$500K-$1M'
    WHEN total_revenue < 2000000 THEN '$1M-$2M'
    WHEN total_revenue < 3500000 THEN '$2M-$3.5M'
    WHEN total_revenue < 5000000 THEN '$3.5M-$5M'
    ELSE '>$5M'
END;

-- Grant dependency bands
UPDATE f990_2025.prospects
SET grant_dep_band = CASE
    WHEN grant_dependency_pct < 25 THEN '<25%'
    WHEN grant_dependency_pct < 50 THEN '25-50%'
    WHEN grant_dependency_pct < 75 THEN '50-75%'
    WHEN grant_dependency_pct < 90 THEN '75-90%'
    ELSE '90-100%'
END;

-- Sector (first letter of NTEE)
UPDATE f990_2025.prospects
SET sector = LEFT(ntee_code, 1)
WHERE ntee_code IS NOT NULL;
```

## Step 6: Calculate ICP Score

Scoring model (0-15 points):

| Factor | Points | Condition |
|--------|--------|-----------|
| Revenue sweet spot | +3 | $1M - $3.5M |
| Revenue acceptable | +1 | $500K-$1M OR $3.5M-$5M |
| High grant dependency | +2 | 50%+ |
| Underserved by foundations | +3 | 0-3 grants received |
| Somewhat underserved | +1 | 4-5 grants received |
| Proven sector | +2 | NTEE starts with E, P, B, or N |
| California | +1 | state = 'CA' |
| Has employees | +1 | employee_count BETWEEN 5 AND 50 |
| Growing (if calculable) | +1 | yoy_revenue_growth > 0 |

```sql
UPDATE f990_2025.prospects
SET icp_score = 
    -- Revenue sweet spot
    CASE 
        WHEN total_revenue BETWEEN 1000000 AND 3500000 THEN 3
        WHEN total_revenue BETWEEN 500000 AND 1000000 THEN 1
        WHEN total_revenue BETWEEN 3500000 AND 5000000 THEN 1
        ELSE 0
    END
    -- High grant dependency
    + CASE WHEN grant_dependency_pct >= 50 THEN 2 ELSE 0 END
    -- Underserved by foundations
    + CASE 
        WHEN num_foundation_grants <= 3 THEN 3
        WHEN num_foundation_grants <= 5 THEN 1
        ELSE 0
    END
    -- Proven sector
    + CASE WHEN sector IN ('E', 'P', 'B', 'N') THEN 2 ELSE 0 END
    -- California
    + CASE WHEN state = 'CA' THEN 1 ELSE 0 END
    -- Has employees
    + CASE WHEN employee_count BETWEEN 5 AND 50 THEN 1 ELSE 0 END;
```

## Step 9: Output Summary Stats

After population, report:

```sql
-- Total prospects
SELECT COUNT(*) as total_prospects FROM f990_2025.prospects;

-- By priority tier
SELECT priority_tier, COUNT(*) as count
FROM f990_2025.prospects
GROUP BY priority_tier
ORDER BY priority_tier;

-- By state (top 10)
SELECT state, COUNT(*) as count
FROM f990_2025.prospects
GROUP BY state
ORDER BY count DESC
LIMIT 10;

-- By sector
SELECT sector, COUNT(*) as count
FROM f990_2025.prospects
GROUP BY sector
ORDER BY count DESC;

-- By revenue band
SELECT revenue_band, COUNT(*) as count
FROM f990_2025.prospects
GROUP BY revenue_band
ORDER BY revenue_band;

-- By ICP score distribution
SELECT icp_score, COUNT(*) as count
FROM f990_2025.prospects
GROUP BY icp_score
ORDER BY icp_score DESC;

-- Top 20 prospects
SELECT ein, org_name, state, sector, total_revenue, 
       num_foundation_grants, icp_score, priority_tier
FROM f990_2025.prospects
ORDER BY icp_score DESC
LIMIT 20;
```
