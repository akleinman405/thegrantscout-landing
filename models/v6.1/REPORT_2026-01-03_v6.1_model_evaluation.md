## The Modeling Journey: V1 to V6.1

### The Problem We're Solving

Given a nonprofit client, rank ~140,000 private foundations by likelihood to fund them. The goal is **discovery** - finding foundations the client doesn't already know about that are genuinely likely to fund their work.

### V1-V3: Early Explorations

| Version | Approach | Issue |
|---------|----------|-------|
| V1 | Simple heuristics (state match, sector match) | No learning, arbitrary weights |
| V2 | Random Forest classifier | Overfitting, hard to interpret |
| V3 | Gradient Boosted Trees | Better but still black-box |

**Key Learning:** We needed an interpretable model where we could understand and validate what it learned.

### V4: First LASSO Model

Switched to LASSO (L1-regularized) logistic regression because:
- **Interpretable:** Coefficients show exactly what matters
- **Automatic feature selection:** Irrelevant features get zero coefficients
- **Generalizable:** Regularization prevents overfitting

**V4 Results:**
- AUC: 0.89
- Top features: state match, sector match, foundation size
- Issue: Semantic similarity had wrong sign (negative coefficient)

### V5: Added Semantic Similarity

Added sentence-transformer embeddings (all-MiniLM-L6-v2) to capture mission alignment:
- Foundation average grant purpose embedding
- Recipient mission embedding
- Cosine similarity between them

**V5 Results:**
- AUC: 0.91
- Semantic similarity coefficient: -0.15 (still wrong sign!)

**Root Cause Analysis:** The training data was biased. Negatives were random foundation-recipient pairs, which happened to have larger recipients on average. The model learned "big orgs get grants" rather than "similar missions get grants."

### V6: Attempted Fix

Tried to fix by adding more features and re-weighting.

**V6 Results:**
- AUC: 0.98 (suspiciously high!)
- Recipient revenue coefficient: +4.07 (dominated everything)
- Semantic similarity: -0.31 (still wrong!)

**Diagnosis:** Simpson's Paradox. The model learned that recipient size predicts grants because our negative sampling was biased toward small recipients.

### V6.1: Size-Matched Negative Sampling (Current)

**The Fix:** Instead of random negatives, create negatives matched by recipient size bucket.

```
For each positive (foundation → recipient with $X revenue):
  Sample a negative (same foundation → different recipient with similar $X revenue)
```

This forces the model to learn "within size class, what predicts a grant?" rather than "big orgs get grants."

## V6.1 Design Choices

### Choice 1: Size Buckets for Matching

| Bucket | Revenue Range | Why This Split |
|--------|---------------|----------------|
| Tiny | <$500K | Small community orgs |
| Small | $500K-$2M | Growing nonprofits |
| Medium | $2M-$10M | Established organizations |
| Large | >$10M | Major institutions |

**Rationale:** These buckets reflect natural breaks in nonprofit scale and foundation giving patterns.

### Choice 2: LASSO Regularization (lambda = 1se)

Used `lambda.1se` (one standard error rule) rather than `lambda.min`:
- More regularization = fewer features = more generalizable
- Prevents overfitting to training quirks
- Final model has ~60 non-zero coefficients (down from 100+)

### Choice 3: Semantic Similarity with Actual Embeddings

Previous versions used sector-average embeddings as fallback too often. V6.1 prioritizes:
1. **Actual recipient mission embedding** (91% coverage)
2. **Sector average embedding** (9% fallback)

This gives more precise mission alignment scores.

### Choice 4: Logit-Based Ranking (Not Probability)

**Problem Discovered:** Sigmoid function saturates - all probabilities were 99-100%.

**Solution:** Use raw logit scores for ranking. The logit preserves full discrimination:
```
logit = intercept + Σ(coefficient × scaled_feature)
probability = sigmoid(logit)  # Saturates near 1.0
ranking_score = logit  # Full range, good discrimination
```

### Choice 5: Captive Foundation Filter

**Problem:** 15% of foundations only give to 1 recipient (e.g., family trusts that only fund one specific org).

**Solution:** Filter `unique_recipients_all_time >= 5`

**Impact:** Removes 3,878 captive foundations that would never fund a new client.

### Choice 6: Preselected-Only Filter

**Problem:** Some foundations mark `preselected_only = TRUE` on their 990-PF, meaning they don't accept applications.

**Solution:** Filter `preselected_only = FALSE OR preselected_only IS NULL`

**Impact:** Removes 1,082 additional foundations, ensuring all recommendations are accessible.

## Known Limitations & Future Work

### 1. Semantic Similarity Has Low Weight
The coefficient (+0.04) is small compared to state/sector match (+1.69/+1.60). High semantic matches sometimes rank below foundations with strong geographic patterns.

**Mitigation:** Flag matches with semantic_sim > 0.4 as "mission-aligned" in reports.

**Future:** Consider V6.2 with boosted semantic weight or hybrid scoring.

### 2. Some Irrelevant Foundations Still Rank High
Foundations with strong state match but low semantic fit can rank highly.

**Example:** Elizabeth Carse Foundation (#3 for HN with 0.07 semantic sim)

**Mitigation:** Review top 20 manually before sending reports.

### 3. Model Doesn't Know Foundation Priorities
A foundation might have shifted focus (e.g., COVID pivot) that isn't reflected in historical data.

**Future:** Add web-scraped foundation priority signals.

## Glossary

| Term | Definition |
|------|------------|
| LASSO | Least Absolute Shrinkage and Selection Operator - regularized regression |
| AUC | Area Under ROC Curve - model discrimination metric (0.5 = random, 1.0 = perfect) |
| Logit | Log-odds, the linear predictor before sigmoid transformation |
| Semantic similarity | Cosine similarity between mission embeddings (0-1 scale) |
| Captive foundation | Foundation that only funds 1-2 specific organizations |
| Preselected only | Foundation that doesn't accept unsolicited applications |
| Openness score | Metric (0-1) of how "open" a foundation is to new grantees |
| Size bucket | Revenue category (tiny/small/medium/large) for balanced sampling |
