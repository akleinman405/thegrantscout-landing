## Project 1: URL Enrichment Pipeline

**Goal:** Build a 3-step pipeline to discover and validate website URLs for all 813K orgs (foundations + nonprofits) in the database.

### How Org Websites Are Discovered

The URL enrichment pipeline has 3 stages, run sequentially:

#### Step 1: Clean & Normalize (`01_clean_org_urls.py`)
- Input: `org_url_enrichment` table (813,698 rows — 143K foundations + 670K nonprofits)
- Sources raw website URLs from IRS 990 filings (`website_address_txt` field)
- Detects junk values (~60 patterns: N/A, NONE, phone numbers, aggregator domains)
- Extracts social media URLs (Facebook, LinkedIn) to separate columns
- Normalizes real URLs: lowercase, add https://, strip tracking params
- Result: **377,175 real URLs** (46.4%) + 435K junk removed

#### Step 2: Validate URLs (`02_validate_org_urls.py`)
- Makes async HTTP HEAD requests to all 377K cleaned URLs (50 concurrent)
- GET fallback when HEAD fails
- Classifies: alive (200-399), bot-blocked (403/405/429/503 = still alive), dead (DNS failure, 404)
- Captures redirect targets (final URL)
- Checkpoint/resume via state file
- Result: **242,564 validated alive** (64%), 55,936 dead, 78,701 pending

#### Step 3: Search for Missing URLs (`03_search_org_urls.py`)
- For orgs without IRS-filed websites, searches DuckDuckGo
- 3 parallel workers with rate limiting and iterative backoff
- Confidence scoring: domain-name matching, acronym matching, snippet verification
- Filters 100+ aggregator/directory domains (GuideStar, Charity Navigator, etc.)
- Test run (100 foundations): 25% hit rate, 100% high confidence
