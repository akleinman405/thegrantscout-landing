## 3. Export Local Database

### Step 3.1: Create Export Directory

```bash
mkdir -p ~/supabase-migration
cd ~/supabase-migration
```

### Step 3.2: Export Full Database

```bash
# This may take 15-30 minutes for ~20GB
pg_dump -h localhost -U postgres -d thegrantscout \
  --no-owner \
  --no-acl \
  --format=custom \
  --file=thegrantscout_backup.dump

# Check file size
ls -lh thegrantscout_backup.dump
```

**Expected output:** 5-15GB file (compressed)

### Step 3.3: Alternative - Schema Only First (Optional)

If you want to test with schema first:

```bash

pg_dump -h localhost -U postgres -d thegrantscout \
  --schema-only \
  --no-owner \
  --no-acl \
  --file=thegrantscout_schema.sql
```

### Step 4.3: Restore Database

```bash
# Full restore (may take 1-2 hours for large database)
pg_restore \
  --host=db.xxxxxxxxxxxx.supabase.co \
  --port=5432 \
  --username=postgres \
  --dbname=postgres \
  --no-owner \
  --no-acl \
  --verbose \
  ~/supabase-migration/thegrantscout_backup.dump

# Enter password when prompted
```

### Step 4.4: Monitor Progress

In another terminal, you can check table counts:


## 9. Troubleshooting

### Connection Refused

```
psql: error: connection refused
```

**Fix:** Check Supabase dashboard → Project Settings → Database → Connection string is correct

### Permission Denied

```
ERROR: permission denied for schema f990_2025
```

**Fix:** Run in Supabase SQL Editor:
```sql
GRANT ALL ON SCHEMA f990_2025 TO postgres;
GRANT ALL ON ALL TABLES IN SCHEMA f990_2025 TO postgres;
```

### pg_restore Errors

```
pg_restore: error: could not execute query
```

**Common causes:**
- pgvector extension not enabled → Enable it first
- Role doesn't exist → Use `--no-owner` flag
- Try with `--clean` flag to drop objects first

### Slow Queries

Supabase may need indexes recreated:

```sql
REINDEX SCHEMA f990_2025;
ANALYZE;
```

### SSL Required

If connection fails with SSL error:

```bash
# Add ?sslmode=require to connection string
psql "postgresql://postgres:PASSWORD@db.xxx.supabase.co:5432/postgres?sslmode=require"
```
