Decision 5: Use existing scripts or new approach?

The existing scripts (campaign_manager.py, generate_emails.py, send_generated_emails.py) work but have limitations:

  • Templates are static strings in config.py (no AI generation)
  • Scripts read from grantscout_campaign tables (not fp2/np2)
  • No "give first" data integration (real foundation names/amounts in body)

Recommendation for Week 1: Write emails manually in Claude Code, insert into generated_emails table, send via send_generated_emails.py. This bypasses the template limitation while using the proven sending infrastructure.

For Week 4+: Build a new generate_emails_v2.py that uses Anthropic API + prospect profile views.

4. Phase 2: Build Prospect Profiles 1-2 hours

For each prospect, we need to generate the "give first" data that goes in the email body.

Step 2.1: Foundation Profiles (New Grantee Discovery)

For each foundation, find 3-5 nonprofits that match their giving pattern but haven't received their funding.

Run this query once per foundation, replacing :fdn_ein with their EIN.
SQL: New Grantee Discovery (per foundation) 
-- Replace :fdn_ein with the foundation's EIN
WITH fdn_profile AS (
  SELECT
    sector_focus,
    geographic_focus,
    median_grant
  FROM f990_2025.calc_foundation_profiles
  WHERE ein = ':fdn_ein'
)
SELECT
  nr.organization_name,
  nr.state,
  nr.city,
  COALESCE(np2.mission_description, 'N/A') as mission,
  nr.total_revenue,
  nr.contributions_grants as foundation_grants_received
FROM f990_2025.nonprofit_returns nr
JOIN f990_2025.nonprofits_prospects2 np2 ON nr.ein = np2.ein
CROSS JOIN fdn_profile fp
WHERE nr.tax_year = (SELECT MAX(tax_year) FROM f990_2025.nonprofit_returns WHERE ein = nr.ein)
  AND LEFT(np2.ntee_code, 1) IN (
    SELECT key FROM f990_2025.calc_foundation_profiles,
    jsonb_each_text(sector_focus)
    WHERE ein = ':fdn_ein' AND value::numeric > 0.1
  )
  AND np2.state IN (
    SELECT key FROM f990_2025.calc_foundation_profiles,
    jsonb_each_text(geographic_focus)
    WHERE ein = ':fdn_ein' AND value::numeric > 0.1
  )
  AND np2.ein NOT IN (
    SELECT recipient_ein FROM f990_2025.fact_grants
    WHERE foundation_ein = ':fdn_ein'
  )
  AND nr.total_revenue BETWEEN 500000 AND 50000000
ORDER BY nr.total_revenue DESC
LIMIT 5;

Step 2.2: Nonprofit Profiles (Peer Funder Discovery)

For each nonprofit, find 3-5 foundations that fund similar orgs in their state but have never funded them.

Run this query once per nonprofit, replacing :np_ein, :np_state, and :np_sector.
SQL: Peer Funder Discovery (per nonprofit) 
-- Replace :np_ein, :np_state, :np_sector (single letter NTEE)
SELECT
  fp.foundation_name,
  fp.state as fdn_state,
  cfp.median_grant,
  ROUND(cfp.total_giving_5yr / 1e6, 1) as giving_5yr_m,
  cfp.giving_trend,
  cfp.openness_score,
  COUNT(DISTINCT fg.recipient_ein) as similar_orgs_funded
FROM f990_2025.fact_grants fg
JOIN f990_2025.dim_recipients dr ON fg.recipient_ein = dr.ein
JOIN f990_2025.foundation_prospects2 fp ON fg.foundation_ein = fp.ein
JOIN f990_2025.calc_foundation_profiles cfp ON fp.ein = cfp.ein
WHERE LEFT(dr.ntee_code, 1) = ':np_sector'
  AND dr.state = ':np_state'
  AND fg.tax_year >= 2020
  AND cfp.accepts_applications = true
  AND cfp.total_giving_5yr > 100000
  AND fg.foundation_ein NOT IN (
    SELECT foundation_ein FROM f990_2025.fact_grants
    WHERE recipient_ein = ':np_ein'
  )
GROUP BY fp.foundation_name, fp.state, cfp.median_grant,
  cfp.total_giving_5yr, cfp.giving_trend, cfp.openness_score
ORDER BY cfp.total_giving_5yr DESC
LIMIT 5;

Step 3.3: Write Follow-up Sequences

Each prospect gets a 3-touch sequence. Follow-ups bring NEW value, never "just checking in."

TouchDayContent
Touch 1 (initial)Day 0Peer Funder / New Grantee data (3 names)
Touch 2Day 3Different angle: add 2-3 more names, or switch to sector trend data
Touch 3 (breakup)Day 10Offer the resource one more time, mention it's free, close gracefully
Template: Follow-up Touch 2 (Nonprofit)
Subject: Re: 3 foundations funding [sector] work in [state]
Hi [First Name], One more thing: foundation giving to [sector] in [state] totaled $[X]M in 2024. Here are 2 more active funders I found: - [Foundation D]: $[X]M in 5-year giving, accepts unsolicited proposals - [Foundation E]: recently started funding [sector], median $[X]K Happy to share application details for any of these. No cost. Alec
Template: Follow-up Touch 3 / Breakup (Nonprofit)
Subject: Re: 3 foundations funding [sector] work in [state]
Hi [First Name], Last note on this. I put together a short list of foundations actively funding [sector] work like yours. It's free and I'll send it over if you'd like. If the timing isn't right, no worries at all. Alec
What NOT to write (crisis exploitation examples):
  • "With federal funding cuts threatening organizations like yours..." (The Fear Amplifier)
  • "We noticed your government grants were $X..." (The Voyeur)
  • "In these uncertain times..." (The Ambulance Chaser)
  • "Many nonprofits are scrambling to find..." (The Panic Pusher)

If government cuts come up, only reference as context in follow-ups, never first touch. Use: "the current funding environment" not "devastating crisis."

Step 3.4: Review Ethics Checklist

  • Does each email lead with a solution, not a wound?
  • Does NO email reference specific funding cuts?
  • Is the value unconditional (useful whether they subscribe or not)?
  • Would this feel like solidarity if read on their worst day?
  • Is every data point verifiable from public 990 filings?
  • Are all foundation/nonprofit names accurate (double-check against DB)?
  • Is each email 50-80 words?

8. Phase 6: Systematize (Nightly Batch) Week 4+

Once manual emails are working and reply rates are validated, automate the pipeline.

The Nightly Batch Process

  1. 7 PM: Query next day's cohort (~50 prospects, same state+sector+budget)
  2. 7:30 PM: Run profile queries for each prospect (peer funders / new grantees)
  3. 8 PM: Anthropic API writes personalized emails using profile data
  4. 8:30 PM: Insert into generated_emails table as 'pending'
  5. Next day 9 AM: send_generated_emails.py sends during business hours
  6. Throughout day: Auto-queue Day 3 and Day 10 follow-ups for non-repliers

B. Ethics Framework (7 Rules)
  1. Lead with the solution, not the wound. "We identified 5 foundations actively funding [sector]" not "We know your federal funding was cut."
  2. Never reference specific cuts unless the recipient mentioned them first.
  3. Make the value unconditional. The free insight is useful whether or not they subscribe.
  4. Price the service the same as before the crisis. No "emergency packages."
  5. Acknowledge context with restraint. "The current funding environment" not "devastating crisis."
  6. Pass the "worst day" test. If they read this after laying off staff, does it feel like solidarity?
  7. Offer to help even if they cannot pay. "If budget is a concern, reply and we'll share what we can at no cost."

  cfp.trend_pct_change,
  cfp.openness_score,
  cfp.repeat_rate,
  cfp.new_recipients_5yr,

  cfp.geographic_focus,
