## Executive Summary

We have 143,184 foundations in our database. After analyzing our existing data, the situation is better than the 9.9% figure suggested: **80,930 foundations (56.5%) already have validated URLs**. The remaining 62,254 without validated URLs break down as follows:

- ~58,021 have no usable URL in any of their 990-PF filings (they reported "N/A", "none", or left it blank)
- ~4,233 have a URL that was extracted but failed HTTP validation

The critical insight is that most of the 58,000 foundations without URLs genuinely do not have websites. They reported "N/A" on their own tax filings. No external data source will produce URLs for foundations that do not have websites.

For the **production pipeline** (assets >= $100K, accepts applications, grants to orgs, >= 5 unique recipients), only **66 foundations out of 474 (13.9%)** lack validated URLs. This is a manageable number that could be resolved through manual research or targeted web scraping.

The most promising external sources for filling gaps are: (1) the Giving Tuesday 990 Data Lake for catching any XML filings we missed, (2) Candid's Premier API for the highest-value missing foundations, and (3) targeted Google/DuckDuckGo search for the ~95 production-viable foundations lacking URLs.

## Source 1: ProPublica Nonprofit Explorer

**What it is:** ProPublica's free searchable database of nonprofit tax filings, with an API and links to raw IRS data.

**Does it have foundation URLs?** No.

**Coverage:** 1.9M+ nonprofit records, including private foundations.

**Cost:** Free (API and bulk data).

**Access method:** REST API (v2) at `https://projects.propublica.org/nonprofits/api`. Bulk data links to the same IRS XML files we already process.

**Data freshness:** Updated as IRS releases new filings.

**Key fields available:** EIN, name, address, NTEE code, subsection code, total revenue, total expenses, assets, liabilities, GuideStar URL, NCCS URL. No organization website URL field in the API response. The bulk data is just links to raw IRS XML filings.

**Practical assessment:** Not useful for URL discovery. The API explicitly lacks a website field. The "bulk data" is just the same IRS XML files we already process. We have already extracted `WebsiteAddressTxt` from all 638,082 of our 990-PF XML filings. ProPublica adds no incremental value for URLs.

## Source 2: IRS Business Master File (EO BMF)

**What it is:** The IRS master registry of all tax-exempt organizations, published monthly as CSV files by state.

**Does it have foundation URLs?** No.

**Coverage:** All tax-exempt organizations (~1.8M records). Updated monthly (2nd Monday).

**Cost:** Free.

**Access method:** CSV download by state from [IRS EO BMF page](https://www.irs.gov/charities-non-profits/exempt-organizations-business-master-file-extract-eo-bmf).

**Data freshness:** Monthly updates.

**Fields available:** EIN, organization name, address (street/city/state/zip), group exemption number, subsection code, affiliation code, classification codes, ruling date, deductibility code, foundation code, activity codes, organization code, status code, tax period, asset code/amount, income code/amount, revenue amount, NTEE code, filing requirement code, accounting period.

**Practical assessment:** Not useful for URL discovery. The BMF is a registration/status file. It has no website URL field. It could be useful for other purposes (verifying active status, getting updated addresses, NTEE codes) but does not help with URLs.

## Source 4: Candid (GuideStar + Foundation Directory)

**What it is:** The largest aggregator of nonprofit and foundation data in the US, formed from the merger of GuideStar and Foundation Center. Offers APIs, Foundation Directory Online (FDO), and Foundation Maps.

**Does it have foundation URLs?** Yes. The Premier API v3 includes a `website_url` field.

**Coverage:** 1.6M+ nonprofits (all types, not just foundations). Foundation Directory covers ~140K+ US grantmakers.

**Cost:**

| Product | Annual Cost |
|---------|------------|
| Essentials API | $4,800/yr |
| Premier API (includes website_url) | $9,900/yr |
| Grants API | $6,000/yr |
| Foundation Directory Essential | ~$427/yr ($35.54/mo) |
| Foundation Directory Professional | ~$1,519/yr ($126.58/mo) |
| Foundation Directory Enterprise | Custom quote |
| Demographics API | Free |
| Taxonomy API | Free |

**Access method:** REST APIs via [developer.candid.org](https://developer.candid.org/). Foundation Directory Online via web interface. Free 30-day trial available. API responses include JSON data.

**Data freshness:** Continuously updated. Candid aggregates data from IRS filings, self-reported profiles (GuideStar Seal of Transparency), and their own research.

**Key finding:** The Premier API v3 response explicitly includes `"website_url": "candid.org"` (example shown in their docs). It also includes `social_media_urls`, `donation_page`, and logo URLs. The Premier API includes 760+ fields per organization.

**Practical assessment:** This is the single richest external source for foundation URLs. At $9,900/year for the Premier API, it is expensive for a startup. However, for the ~66-95 production-viable foundations missing URLs, it would be overkill. The Foundation Directory Professional subscription ($1,519/yr) might provide website data through the web interface for manual lookup. A 30-day free trial of the API could let us batch-query the missing foundations. Worth evaluating if we need URLs at scale beyond what we already have.

**Practical assessment:** Not useful for URL discovery. State registries are designed for donor protection and compliance verification, not as data sources. The effort to scrape individual lookups from multiple state websites would be significant for minimal return.

## Source 7: IRS 990 E-File XML Bulk Data

**What it is:** The raw XML files of electronically filed 990, 990-EZ, and 990-PF returns, made available by the IRS.

**Does it have foundation URLs?** Yes. The 990-PF XML schema includes a `WebsiteAddressTxt` element (Line J of the form header). This is the exact field we already extract.

**Coverage:** All e-filed 990-PF returns. Since 2020, the IRS mandated e-filing for most 990-PF filers, so coverage is near-complete for recent years.

**Cost:** Free.

**Access methods (multiple):**

| Source | Status | URL |
|--------|--------|-----|
| IRS Form 990 Series Downloads | Active, current | [irs.gov/form-990-series-downloads](https://www.irs.gov/charities-non-profits/form-990-series-downloads) |
| Giving Tuesday 990 Data Lake | Active, curated | [990data.givingtuesday.org](https://990data.givingtuesday.org/) |
| AWS S3 (original IRS bucket) | Discontinued Dec 2021 | [registry.opendata.aws/irs990](https://registry.opendata.aws/irs990/) |
| Nonprofit Open Data Collective | Active, tools + index | [GitHub](https://github.com/Nonprofit-Open-Data-Collective) |

**Data freshness:** IRS releases new filings monthly. Giving Tuesday Data Lake updates regularly.

**Key tools for extraction:**

| Tool | Description |
|------|-------------|
| IRSx (990-xml-reader) | Python library to extract fields from 990 XML by xpath. Supports 2013+ filings. |
| form-990-xml-mapper | Generates data dictionary from IRS XML schemas (by Giving Tuesday) |
| form-990-xml-parser | Command-line XML parser (by Giving Tuesday) |
| Master Concordance File | CSV mapping ~10,000 xpaths to standardized variable names |
| Giving Tuesday Data Marts | Pre-parsed, research-ready extracts of 990/990-PF/990-EZ data |

**The WebsiteAddressTxt xpath:** Confirmed present in the 990-PF XML schema. Element name `WebsiteAddressTxt`, description "Web Site", Line J. It is defined as optional (minOccurs=0), which explains why many foundations leave it blank or write "N/A".

**Practical assessment:** This is the canonical source, and we already use it. All 638,082 of our pf_returns records come from XML e-files, and we already extract the `website_url` field from them. The ~58,000 foundations without URLs genuinely reported "N/A" or left it blank on their filings. The Giving Tuesday Data Lake is worth checking to see if they have any filings we missed (e.g., newer filings where a foundation added a website). The curated 990-PF data marts could provide a quick cross-check.

### Key Insight

The real problem is not data source availability. The problem is that **most small private foundations do not have websites**. Of the 62,254 foundations lacking validated URLs, approximately 44,000 explicitly reported "N/A" on their tax returns, and another 14,000+ left the field blank or entered junk values ("none", "not applicable", "under construction", etc.). These are family foundations, donor-advised funds, and small trusts that operate through banks and attorneys without any web presence.

For the **production pipeline**, the URL gap is small (66-95 foundations) and addressable through targeted web search and the Candid trial. The investment in expensive external data sources is not justified by the marginal gain.
