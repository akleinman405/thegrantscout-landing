## Executive Summary

We evaluated 5 existing cold email angles and brainstormed 15 new ones (7 nonprofit, 8 foundation), running 40+ SQL queries against production data to verify feasibility. Key findings:

- **Strongest nonprofit angles** lean on peer comparison and new funder discovery, not crisis messaging. Our database can identify 451,823 nonprofits with identifiable peer groups and 209,154 nonprofits that already appear as grant recipients, enabling "here's what your peers are getting funded for" messaging at massive scale.
- **Strongest foundation angles** exploit co-funder network analysis and grantee portfolio health monitoring. We verified co-funder analysis works (tested with real EINs, found networks of 4,900+ shared grantees between major funders) and can track grantee revenue trends across 27,000+ organizations per large foundation.
- **Government funding cut angle** is powerful but time-limited. 103,653 nonprofits have government grants in our data, with 35,974 majority-dependent on government funding. However, this angle carries the highest ethical risk and shortest shelf life.
- **Board connection angle** is theoretically powerful (355,165 distinct foundation officers, 623,290 nonprofits with officer data) but requires expensive name-matching compute that is hard to scale in batch.

## Evaluation of 5 Existing Angles

### Angle 1: State/Sector Foundation List (Nonprofit)

**Pitch:** "We're putting together a list of the top foundations funding nonprofits in [your state/sector]. Want a copy?"

**Data Feasibility:**

```sql
-- 149 viable state+sector combinations with 10+ active, application-accepting foundations
SELECT fp.state, LEFT(fp.bmf_ntee_cd, 1) as sector, COUNT(*) as foundation_count
FROM f990_2025.foundation_prospects2 fp
JOIN f990_2025.calc_foundation_profiles cp ON fp.ein = cp.ein
WHERE fp.state IS NOT NULL AND fp.bmf_ntee_cd IS NOT NULL
  AND cp.has_grant_history = true AND cp.accepts_applications = true
GROUP BY fp.state, LEFT(fp.bmf_ntee_cd, 1)
HAVING COUNT(*) >= 10;
-- Result: 149 state/sector combos
```

Additional data per foundation in each list:
```sql
-- For each foundation: name, assets, median grant, 5yr giving, trend, repeat rate, openness
SELECT fp.foundation_name, fp.state, fp.total_assets_eoy_amt,
  cp.median_grant, cp.total_giving_5yr, cp.giving_trend, cp.repeat_rate, cp.openness_score
FROM f990_2025.foundation_prospects2 fp
JOIN f990_2025.calc_foundation_profiles cp ON fp.ein = cp.ein
WHERE fp.state = '[STATE]' AND LEFT(fp.bmf_ntee_cd, 1) = '[SECTOR]'
  AND cp.has_grant_history = true AND cp.accepts_applications = true
ORDER BY cp.total_giving_5yr DESC;
```

| Dimension | Score | Rationale |
|-----------|-------|-----------|
| Data availability | 5 | State, NTEE, assets, giving metrics all well-populated. 149 viable combos. |
| Scalability | 5 | Pure SQL generation. One query per state/sector combo. Could generate all 149 lists in minutes. |
| Upsell path | 4 | List is generic; upsell to "which of these 50 foundations are best for YOUR specific mission?" is natural. |
| Ethical risk | 1 | Purely positive, informational. No crisis exploitation. |

### Angle 2: Government Funding Cut Alert (Nonprofit)

**Pitch:** "We noticed you were receiving funding from [specific government program] that's been cut. We found a few foundations worth looking into: [Name A], [Name B], [Name C]."

**Data Feasibility:**

```sql
-- 103,653 nonprofits with government grants (tax_year >= 2021)
-- 35,974 with government grants as majority (>50%) of revenue
-- 69,370 with government grants > $100K
SELECT COUNT(*) FROM f990_2025.nonprofit_returns
WHERE government_grants_amt > 0 AND tax_year >= 2021;
```

We can identify government-dependent nonprofits by sector:
```sql
-- Top vulnerable sectors (by count of govt-funded NPs):
-- P (Human Services): 13,877
-- B (Education): 11,293
-- A (Arts): 9,405
-- E (Health): 6,382
-- L (Housing): 5,544
SELECT LEFT(np.ntee_code, 1) as sector, COUNT(DISTINCT nr.ein) as vulnerable_nps
FROM f990_2025.nonprofit_returns nr
JOIN f990_2025.nonprofits_prospects2 np ON nr.ein = np.ein
WHERE nr.government_grants_amt > 0 AND nr.tax_year >= 2021 AND np.ntee_code IS NOT NULL
GROUP BY LEFT(np.ntee_code, 1) ORDER BY vulnerable_nps DESC;
```

Matching to foundations:
```sql
-- For a vulnerable nonprofit, find foundations funding similar orgs in same state/sector
SELECT DISTINCT fp.foundation_name, cp.median_grant, cp.giving_trend
FROM f990_2025.fact_grants fg
JOIN f990_2025.dim_recipients dr ON fg.recipient_ein = dr.ein
JOIN f990_2025.foundation_prospects2 fp ON fg.foundation_ein = fp.ein
JOIN f990_2025.calc_foundation_profiles cp ON fp.ein = cp.ein
WHERE LEFT(dr.ntee_code, 1) = '[NP_SECTOR]' AND dr.state = '[NP_STATE]'
  AND fg.tax_year >= 2020 AND cp.accepts_applications = true
ORDER BY cp.total_giving_5yr DESC LIMIT 3;
```

**Limitation:** We know the nonprofit receives government grants and how much, but we do NOT know which specific federal program they receive from (e.g., HHS, USAID, DOE). The "specific government program that's been cut" claim requires either (a) importing the 26,257 vulnerability scores from our research files, or (b) inferring from NTEE sector + program descriptions.

| Dimension | Score | Rationale |
|-----------|-------|-----------|
| Data availability | 3 | We know WHO gets govt money and HOW MUCH, but not FROM WHICH PROGRAM. Matching to specific cuts requires enrichment. |
| Scalability | 3 | Base identification is SQL (103K NPs). But matching to specific cut programs requires NLP on program descriptions or manual tagging. |
| Upsell path | 5 | Crisis creates urgency. "We found 3 foundations" naturally leads to "we can find 50 more and monitor ongoing." |
| Ethical risk | 4 | High risk of feeling exploitative. "We noticed you're losing funding" reads as ambulance chasing. Requires very careful tone. |
| **Composite** | **7** | |

### Angle 3: Benchmark Data (Nonprofit)

**Pitch:** "Nonprofits your size in [sector] typically receive grants from X foundations. Here's how your funding profile compares."

**Data Feasibility:**

```sql
-- Sector benchmark data available for 15+ sectors with 5,000+ nonprofits each
-- Example: Human Services (P): 53,795 NPs, avg revenue $3.4M, avg grants received $1.7M
-- Education (B): 63,639 NPs, avg revenue $7.5M, avg grants received $2.4M
SELECT LEFT(np.ntee_code, 1) as sector,
  COUNT(DISTINCT np.ein) as np_count,
  ROUND(AVG(nr.total_revenue)::numeric, 0) as avg_revenue,
  ROUND(AVG(nr.contributions_grants)::numeric, 0) as avg_grants_received
FROM f990_2025.nonprofits_prospects2 np
JOIN f990_2025.nonprofit_returns nr ON np.ein = nr.ein
WHERE np.ntee_code IS NOT NULL AND nr.tax_year >= 2021 AND nr.total_revenue > 0
GROUP BY LEFT(np.ntee_code, 1) HAVING COUNT(DISTINCT np.ein) >= 50;
```

For individual nonprofit comparison:
```sql
-- Count how many foundation grants a specific NP receives vs sector average
-- 209,154 nonprofits in np2 appear as grant recipients since 2020
SELECT COUNT(DISTINCT fg.foundation_ein) as your_foundation_count
FROM f990_2025.fact_grants fg
WHERE fg.recipient_ein = '[NP_EIN]' AND fg.tax_year >= 2020;
```

| Dimension | Score | Rationale |
|-----------|-------|-----------|
| Data availability | 4 | Sector benchmarks are robust. Individual NP comparison works for 209K of 673K NPs (31%). Others show as "0 foundation grants" which is still a valid data point. |
| Scalability | 5 | Sector benchmarks are pre-computable. Individual comparison is a single indexed query per NP. |
| Upsell path | 5 | "You're below average" is the most natural upsell in the catalog. "We can help you close the gap" writes itself. |
| Ethical risk | 2 | Minor risk of feeling judgmental ("you're underperforming"). Frame as opportunity, not criticism. |

### Angle 4: Grantee Impact Brief (Foundation)

**Pitch:** "Three of your grantees -- [Org A], [Org B], [Org C] -- recently lost federal funding from [program]. We put together a brief on replacement funding options for them."

**Data Feasibility:**

```sql
-- For any foundation, we can identify grantees with government funding
-- 57,913 grant recipients have government_grants_amt > 0
-- For a large foundation (e.g., EIN 200721133): 31,811 total grantees, 27,513 match to NP returns
-- We can identify government dependency levels:
--   35,974 nonprofits are majority govt-funded
--   17,336 are significantly govt-funded (25-50%)
SELECT fg.recipient_ein, nr.organization_name, nr.government_grants_amt,
  nr.total_revenue,
  ROUND(nr.government_grants_amt::numeric / NULLIF(nr.total_revenue, 0) * 100, 1) as govt_pct
FROM f990_2025.fact_grants fg
JOIN f990_2025.nonprofit_returns nr ON fg.recipient_ein = nr.ein
WHERE fg.foundation_ein = '[FOUNDATION_EIN]' AND fg.tax_year >= 2021
  AND nr.government_grants_amt > 0 AND nr.tax_year >= 2021
ORDER BY govt_pct DESC LIMIT 10;
```

| Dimension | Score | Rationale |
|-----------|-------|-----------|
| Data availability | 3 | Can identify grantees with govt funding (57,913). Cannot pinpoint WHICH federal program without enrichment. |
| Scalability | 3 | Grantee identification is SQL. "Replacement funding options" requires per-grantee matching (our core pipeline), which is compute-intensive but automatable. |
| Upsell path | 5 | Directly demonstrates our core capability to foundations. "We did this for 3 grantees; we can do it for all of them." |
| Ethical risk | 3 | Lower risk than NP version because we're helping a FUNDER protect their GRANTEES. Still crisis-adjacent. |
| **Composite** | **8** | |

### Angle 5: Grantee Exposure Summary (Foundation)

**Pitch:** "We're researching which foundation grantees were most impacted by recent federal cuts in [sector]. Would a summary of your grantees' exposure be useful?"

**Data Feasibility:**

```sql
-- For a foundation, calculate aggregate grantee exposure to government funding
-- Example for EIN 237049738: grantee revenue trends available
-- growing: 6,712 grantees | stable: 3,409 | declining: 3,719
SELECT
  CASE WHEN nr2.total_revenue > nr1.total_revenue * 1.1 THEN 'growing'
       WHEN nr2.total_revenue < nr1.total_revenue * 0.9 THEN 'declining'
       ELSE 'stable' END as revenue_trend,
  COUNT(*) as grantee_count
FROM f990_2025.fact_grants fg
JOIN f990_2025.nonprofit_returns nr1 ON fg.recipient_ein = nr1.ein AND nr1.tax_year = 2021
JOIN f990_2025.nonprofit_returns nr2 ON fg.recipient_ein = nr2.ein AND nr2.tax_year = 2023
WHERE fg.foundation_ein = '[EIN]' AND fg.tax_year >= 2021
GROUP BY revenue_trend;
```

| Dimension | Score | Rationale |
|-----------|-------|-----------|
| Data availability | 4 | Multi-year revenue data available for 27K+ grantees per large foundation. Sector-level exposure summaries computable. |
| Scalability | 4 | Aggregate stats are SQL queries. Per-foundation report generation is automatable. Works for foundations with 10+ grantees (27,555 foundations). |
| Upsell path | 4 | "Here's the overview; we can provide detailed replacement funding plans for each affected grantee." |
| Ethical risk | 3 | Same as Angle 4 but softer ("exposure summary" vs "lost funding"). Still trades on crisis. |
| **Composite** | **9** | |

### Angle 6: Peer Funder Discovery

**Pitch:** "We identified 47 foundations that fund organizations like yours in [state] but haven't funded you yet. Here are the top 5."

**Data Feasibility:**

```sql
-- 451,823 nonprofits have identifiable peers (same NTEE sector + state) with grant history
-- Example: Human Services in CA has 7,086 matching foundations
-- Pipeline: find foundations funding same-sector/same-state recipients, exclude current funders
SELECT DISTINCT fp.foundation_name, cp.median_grant, cp.total_giving_5yr, cp.giving_trend
FROM f990_2025.fact_grants fg
JOIN f990_2025.dim_recipients dr ON fg.recipient_ein = dr.ein
JOIN f990_2025.foundation_prospects2 fp ON fg.foundation_ein = fp.ein
JOIN f990_2025.calc_foundation_profiles cp ON fp.ein = cp.ein
WHERE LEFT(dr.ntee_code, 1) = '[NP_SECTOR]' AND dr.state = '[NP_STATE]'
  AND fg.tax_year >= 2020 AND cp.accepts_applications = true
  AND fg.foundation_ein NOT IN (
    SELECT foundation_ein FROM f990_2025.fact_grants WHERE recipient_ein = '[NP_EIN]'
  )
ORDER BY cp.total_giving_5yr DESC LIMIT 5;
```

| Dimension | Score | Rationale |
|-----------|-------|-----------|
| Data availability | 5 | 451K nonprofits have peers. 7,086 matching foundations just for Human Services/CA. Rich data per foundation. |
| Scalability | 5 | Pure SQL. One query per nonprofit. Can batch-generate for thousands. |
| Upsell path | 5 | "Here are 5 of 47 foundations. Subscribe to get the full list with contact info and positioning strategy." |
| Ethical risk | 1 | Purely positive. Opportunity-focused. No crisis reference. |

### Angle 7: New Funders Entering Your Space

**Pitch:** "3 new foundations started funding [sector] organizations in [state] this year. Here's who they are and what they're looking for."

**Data Feasibility:**

```sql
-- 116,089 new sector-entrant foundation/sector pairs since 2022
-- 2,349,831 first-time grants since 2022
-- Example: 1,941 new funders entered Education in NY since 2022
SELECT DISTINCT fg.foundation_ein, fp.foundation_name, cp.total_giving_5yr, cp.giving_trend,
  MIN(fg.tax_year) as first_grant_year
FROM f990_2025.fact_grants fg
JOIN f990_2025.dim_recipients dr ON fg.recipient_ein = dr.ein
JOIN f990_2025.foundation_prospects2 fp ON fg.foundation_ein = fp.ein
JOIN f990_2025.calc_foundation_profiles cp ON fp.ein = cp.ein
WHERE LEFT(dr.ntee_code, 1) = '[SECTOR]' AND dr.state = '[STATE]'
  AND fg.tax_year >= 2022
  AND NOT EXISTS (
    SELECT 1 FROM f990_2025.fact_grants fg2
    JOIN f990_2025.dim_recipients dr2 ON fg2.recipient_ein = dr2.ein
    WHERE fg2.foundation_ein = fg.foundation_ein
    AND LEFT(dr2.ntee_code, 1) = '[SECTOR]' AND dr2.state = '[STATE]'
    AND fg2.tax_year < 2022
  )
GROUP BY fg.foundation_ein, fp.foundation_name, cp.total_giving_5yr, cp.giving_trend
ORDER BY cp.total_giving_5yr DESC LIMIT 3;
```

| Dimension | Score | Rationale |
|-----------|-------|-----------|
| Data availability | 5 | 116K new sector entrants. Rich data. Purpose text available (99.9%) to explain what they funded. |
| Scalability | 5 | SQL-driven. The NOT EXISTS subquery is heavier but still batch-feasible. |
| Upsell path | 4 | "New funders are the best time to apply." Natural bridge to "we monitor this weekly for you." |
| Ethical risk | 1 | Purely positive and forward-looking. "New opportunity" framing. |
| **Composite** | **13** | |

### Angle 12: Board Connection Finder

**Pitch:** "One of your board members, [Name], also serves on the board of [Foundation Name], which gave $2M last year to organizations like yours."

**Data Feasibility:**

```sql
-- 623,290 nonprofits have all_officers JSONB data (92.6%)
-- 355,165 distinct foundation officers (2021+)
-- 130,434 foundation EINs have officer data
-- Challenge: name matching across 25M+ officer records requires deduplication
-- Approach: extract NP officer names from JSONB, match against foundation officers table
SELECT DISTINCT o.person_nm as shared_person, fp.foundation_name, cp.total_giving_5yr
FROM f990_2025.officers o
JOIN f990_2025.foundation_prospects2 fp ON o.ein = fp.ein
WHERE UPPER(TRIM(o.person_nm)) IN (
  SELECT UPPER(TRIM(elem->>'name'))
  FROM f990_2025.nonprofits_prospects2 np,
       jsonb_array_elements(np.all_officers) elem
  WHERE np.ein = '[NP_EIN]'
)
AND o.tax_year >= 2021
AND o.person_nm IS NOT NULL AND LENGTH(o.person_nm) > 5;
```

| Dimension | Score | Rationale |
|-----------|-------|-----------|
| Data availability | 4 | Rich data on both sides (92.6% NPs, 355K foundation officers). Name matching has false positives ("JOHN SMITH"). |
| Scalability | 2 | Name matching across 25M records is computationally expensive. Requires pre-computed index or batch processing. False positive filtering needed. |
| Upsell path | 5 | Board connections are the #1 fundraising strategy. "We found a connection you didn't know about" is gold. |
| Ethical risk | 2 | Public data (990 filings). Minor privacy concern about surfacing board service. Frame as "public record." |
| **Composite** | **9** | |
