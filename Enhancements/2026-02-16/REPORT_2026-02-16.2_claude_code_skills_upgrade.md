## Changes Implemented

### 1. Fix Global SOP Hook (Critical)

**Problem:** Global `~/.claude/settings.json` had a `UserPromptSubmit` hook that injected TheGrantScout DB rules (`f990_2025`, `thegrantscout`) into every project, including Filtered.

**Fix:**
- Removed the global hook from `~/.claude/settings.json`
- Created project-specific hook at `.claude/hooks/sop-reminder.sh`
- Added project-level `UserPromptSubmit` hook in `.claude/settings.json`

### 2. Upgrade Project Permissions

**Added auto-approve rules for:**
- `python3`, `pip3` (dev commands)
- `git status/log/diff/add/commit/branch/checkout/stash` (git workflow)
- `gh` (GitHub CLI)
- `pandoc` (document conversion)
- `brew services` (PostgreSQL management)

**Added deny rules for:**
- `.env`, `.pem`, `.key`, `credentials.json`, `Postgresql Info.txt` (secrets)
- `rm -rf`, `git push --force`, `git reset --hard` (destructive)
- `cat */.env`, `cat *.pem`, `cat *.key` (cat bypass of Read deny)

**Cleaned up settings.local.json:**
- Removed `Bash(python3:*)` (now in project settings.json)
- Kept `Bash(PGPASSWORD=... psql:*)` (contains credentials, stays local)

### 5. New `/commit` Command

Smart commit workflow:
- Assesses staged/unstaged changes
- Warns about sensitive files
- Uses conventional commit format with TheGrantScout-specific types (`data:`, `report:`, `config:`)
- Includes `Co-Authored-By` trailer

### 6. New `/pipeline` Command (Adapted from `/deploy`)

Instead of Fly.io deployment (Filtered-specific), created a `/pipeline` command that:

- Reports results with metrics
