## Task 1: Update `.claude/skills/grant-report/SKILL.md`

Replace the step-by-step instructions with this flow:

### Step 1: Read inputs
- Read the input JSON (client profile + foundations with dossiers)
- Validate: confirm 5 foundations present, all have EINs, all have type (PF or PC)
- Print summary: client name, 5 foundation names + EINs

### Step 2: Pull funder snapshots
- Run `scripts/pull_funder_snapshots.py` with the input JSON
- Save raw output to `Enhancements/[DATE]/DATA_[DATE]_[client]_funder_snapshots.json`

### ⏸ STOP 1: Review funder snapshot data
- Print a summary table for each foundation showing: annual giving, grant count, median grant, geographic %, repeat rate, trend, # officers found, application info found (yes/no)
- Flag any issues: missing data, EIN mismatches, zero grants, no officers
- **Wait for Alec to approve before continuing**
- If Alec flags issues, fix and re-run Step 2

### Step 3: Merge dossier + DB data
- Combine the dossier notes (from input JSON) with the DB-pulled snapshots
- For each foundation, identify any gaps (e.g., dossier says "rolling deadline" but DB says specific date — use dossier as authoritative for application details, DB as authoritative for grant statistics)

### Step 4: Write the report
- Follow `references/report_template.md` (v6 template)
- Generate all sections: priority table, time allocation, 5 full foundation profiles, 8-week timeline, quick reference
- Save draft to `Enhancements/[DATE]/REPORT_[DATE]_[client]_grant_opportunities.md`

### ⏸ STOP 2: Review draft report
- Tell Alec the report is ready for review at the file path above
- **Wait for Alec to approve or request edits**
- Apply any edits Alec requests

### Step 5: Finalize
- Save final version to `Enhancements/[DATE]/REPORT_[DATE]_[client]_grant_opportunities.md`
- Write session report (see template below) to `Enhancements/[DATE]/REPORT_[DATE]_[client]_grant_report_session.md`
