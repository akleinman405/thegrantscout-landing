## Executive Summary

Created two reusable Claude Code skills for report generation:

1. **grant-report** - Full monthly grant report generation with a Python script (`pull_funder_snapshots.py`) that queries PostgreSQL for funder metrics, officers, application info, and comparable grantees. Includes the v6 report template.
2. **additional-funders** - Companion document listing 10-20 secondary foundation leads with tier assignments and format guide.

Both skills follow the existing skill pattern (matching `SKILL_client_viability_check.md` and `comprehensive-review` directory structure). The Python script was verified against the actual database schema with 8 column name corrections from the original prompt.

## What Was Built

### grant-report Skill

- **SKILL.md**: 4-step process (Pull snapshots, Merge enrichment, Generate report, Output)
- **scripts/pull_funder_snapshots.py**: Queries `thegrantscout` DB for:
  - Foundation type detection (PF vs PC)
  - Profile data (most recent filing year)
  - Grant metrics: count, total, median, range, annual average
  - Geographic concentration (% in client's state)
  - Repeat funding rate
  - 3-year giving trend (Growing/Stable/Declining with %)
  - Giving style (general support vs program-specific, PF only)
  - Comparable grantees (up to 5, in-state preferred)
  - Officers/directors (most recent year)
  - Application info (deadlines, restrictions, contacts, PF only)
- **references/report_template.md**: Copy of v6 template from Pipeline

### additional-funders Skill

- **SKILL.md**: 4-step process (Select foundations, Pull data, Write document, Output)
- **references/format_guide.md**: Tier-based format (A/B/C) with column definitions and examples

## Notes

- The script only has data for tax years present in the database (currently 2016-2024). Foundations with only 1 year of data show "Insufficient data" for giving trend.
- Giving style analysis depends on purpose text containing keywords like "general", "operating", "unrestricted". Foundations with no purpose text will show null.
- The `is_officer`/`is_director`/`is_trustee` boolean flags appear to be FALSE for all officers in the test case, suggesting these may not be reliably populated in the XML parsing. The `title_txt` field is the reliable source for role classification.
- Skills are NOT registered as Claude Code slash commands. They serve as reference documentation that Claude reads when generating reports. To make them slash commands, they would need to be placed in the user-level skills directory with proper frontmatter.
