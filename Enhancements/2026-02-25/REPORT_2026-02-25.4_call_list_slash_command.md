## Executive Summary

Converted the ad-hoc prospect call list script (3 iterations, hardcoded SQL) into a reusable `/call-list` slash command with:
- JSON config for keywords and presets (single source of truth)
- Shared text-cleaning module (`text_utils.py`)
- CLI tool with `--preview-only`, `--dry-run`, `--list-presets` modes
- Slash command with STOP gate workflow
- Parameterized SQL with safety assertions

Validation: `--preset veterans-national` produces 130 query rows / 100 output rows, matching the session's manual output.

## Architecture

### Config-driven SQL generation

Keywords are defined once in `call_list_keywords.json`. The `build_sql()` function:
1. Iterates categories to build WHERE clause ILIKE conditions
2. Iterates categories (sorted by priority) to build CASE statement
3. Uses a single shared `params` list for both, with assertion check at the end
4. Whitelists `search_fields` against `ALLOWED_FIELDS` to prevent injection

### Presets

| Preset | Categories | State | Limit | State Cap |

| veterans-national | veterans, sailing, aviation, therapy | national | 100 | 15 |
| veterans-ca | veterans, sailing, therapy | CA | 50 | none |
| maritime-national | sailing | national | 50 | 10 |

### CLI precedence

`--preset` loads config, then explicit CLI flags override any preset value.
