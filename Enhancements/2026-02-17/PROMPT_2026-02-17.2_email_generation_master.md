# EMAIL GENERATION PROMPT: Nonprofit Cold Outreach (Give-First Approach)

**Date:** 2026-02-17
**Version:** 2.0
**Source:** REPORT_2026-02-17.1_email_prompt_redesign.md (Section 10), adapted for cohort_foundation_lists_v2 schema

---

## Context

You are drafting personalized cold emails to nonprofit prospects. Each email offers
a curated foundation research set: a collection of private foundations that fund
organizations like theirs in their state and sector.

The emails should be genuinely useful standalone communications, not sales pitches.
The recipient should get value from reading the email even if they never reply.

## Input Data Per Prospect

For each prospect, you will receive fields from `nonprofits_prospects2`:
- `contact_first_name`: First name of the contact
- `contact_email`: Email address
- `organization_name`: Organization name
- `state`: Two-letter state code
- `ntee_code`: Full NTEE code
- `mission_description`: The org's mission statement
- `email_quality_tier`: A/B/C/D quality rating
- `email_quality_flags`: Array of issues (placeholder_name, generic_email, etc.)

Plus cohort context from `cohort_viability`:
- `foundation_count`: Number of foundations in their state+sector cohort
- `display_text`: Pre-computed phrasing ("67 foundations that fund", "over 75 foundations that fund", etc.)

Plus top 25 foundations from `cohort_foundation_lists_v2` with fields:
- `foundation_name`, `foundation_ein`
- `foundation_rank` (1-25, ranked by composite email_fit_score)
- `total_giving_5yr`, `median_grant`, `unique_recipients`, `recent_recipients`
- `last_active_year`, `assets`
- `state_pct` (% of foundation's giving in this state)
- `geo_score`, `size_fit_score`, `openness_score`, `recency_score`, `volume_score`
- `mega_penalty` (1.00/0.75/0.50)
- `email_fit_score` (composite, higher = better fit)
- `geo_tier` (primary/secondary/tertiary)
- `is_national_funder` (boolean)
- `accepts_applications` (boolean)
- `example_recipient_name`, `example_recipient_ein`
- `example_grant_amount`, `example_grant_purpose`, `example_grant_year`
- `example_match_tier` (1-4, see Example Language Rules below)
- `example_recipient_revenue`
- `sector_label` (human-readable: "Human Services", "Education", etc.)

## Email Specifications

- **LENGTH:** 50-90 words in the body (before signature)
- **FORMAT:** Plain text. No links, no images, no HTML formatting, no bold, no bullets.
- **PARAGRAPHS:** 2-4 short paragraphs. Vary the count across the batch.
- **SUBJECT LINE:** 4-7 words. Reference their state or sector. No clickbait, no "list,"
  no "leads," no "free." Include a number when natural.
  Examples: "67 Kansas housing foundations" / "Washington behavioral health funders" /
  "Know about the Sunderland Foundation?"

## Foundation Selection Rules

For each email, select exactly 2 foundations from the prospect's cohort data.
Apply these filters IN ORDER:

1. **GEOGRAPHIC FIT:** `state_pct >= 0.15` OR `geo_tier` is "primary" or "secondary".
   If a foundation gives 95% of its money in Indiana, do not cite it for a New York
   nonprofit, even if it made one grant there.

2. **SCALE FIT:** `median_grant` is between 0.2x and 10x the prospect's typical grant
   need. A foundation giving $5M grants is not relevant to a $400K org.

3. **NICHE FIT:** Read the prospect's `mission_description`. Read the
   `example_recipient_name` and `example_grant_purpose`. The example recipient must do
   recognizably similar work. "Human services" is NOT specific enough. Domestic violence
   services, disability services, refugee services, youth development, and senior services
   are all "human services" but NOT similar work.

4. **DISCOVERY VALUE:** Prefer foundations ranked 3-10 over foundations ranked 1-2.
   The first foundation cited should make the recipient think "I haven't heard of
   that one," not "everyone knows about them."

5. **HARD EXCLUSIONS:**
   - `assets > $5 billion`
   - `mega_penalty = 0.50` (billion-dollar foundations)
   - `accepts_applications = false`
   - Example recipient does different work than the prospect

## Example Language Rules (Based on `example_match_tier`)

The `example_match_tier` column (1-4) tells you how close the example grant match is.
Adjust your language accordingly:

- **Tier 1-2** (exact or close NTEE match): "They recently funded [Name], which does
  similar work to yours, with a $X grant"
- **Tier 3** (same broad sector): "They've supported [Name], a [sector] organization
  in [state], with $X"
- **Tier 4** (fallback, sector + state only): "They've given $X to [sector]
  organizations in [state]" (no "similar work" claim, no specific recipient name)

If the example recipient name looks like it does clearly different work than the
prospect (even at the same tier), omit the name and use the Tier 4 phrasing.

## Display Count Rules (from `cohort_viability.display_text`)

Use the pre-computed `display_text` from the cohort_viability table. This already
handles rounding and capping:
- Under 25 foundations: "several foundations that fund"
- 25-75: exact count rounded to 5 ("65 foundations that fund")
- 76-150: "over 75 foundations that fund"
- 150+: "75 foundations we identified that fund"

Incorporate into the offer phrasing naturally. The `display_text` value ends with
"foundations that fund" so you append the sector and state:
"I put together a breakdown of [display_text] [sector] in [state]."

## Opening Line Strategies

Use each strategy AT LEAST ONCE in a batch of 20. NEVER use any strategy more than
4 times. Assign based on the prospect's situation.

**STRATEGY 1: MISSION ACKNOWLEDGMENT** (for orgs with distinctive missions)
Name what the org does in one specific sentence. Use a detail from mission_description.
"Centro de Apoyo Familiar is one of a few organizations providing bilingual family
support services in the Merrimack Valley."

**STRATEGY 2: SECTOR SHIFT** (for government-funding-dependent orgs)
Reference a behavioral shift, not a crisis.
"A lot of behavioral health organizations in Washington are exploring foundation
funding for the first time."

**STRATEGY 3: GEOGRAPHIC DISCOVERY** (for states with <200 foundations in cohort)
Note something about the foundation landscape in their area.
"Maine has a smaller foundation community, but there are 207 that actively fund
human services there."

**STRATEGY 4: PEER SIGNAL** (when cohort has strong example grant to comparable org)
Reference a peer organization that received a notable grant.
"Heading Home Inc just received $1 million from Liberty Mutual for family shelter
work in Massachusetts."

**STRATEGY 5: FOUNDATION ENTRY** (when a cohort foundation has recent giving trend)
Name a foundation that recently started giving in their space.
"The Satterberg Foundation started funding domestic violence organizations in
Washington in 2022."

**STRATEGY 6: DIRECT RESEARCH STATEMENT** (fallback)
State plainly what you've been researching.
"I've been researching which private foundations fund disability services in
North Carolina."

**STRATEGY 7: QUESTION OPENER** (for orgs with low foundation awareness)
Ask a genuine question about their funding approach.
"Has your team explored private foundation funding for workforce development?"

**NEVER open with:**
- "It's been a tough year" / "These days" / "After everything happening"
- Any generic crisis/uncertainty language
- Any sentence that applies equally to every nonprofit in every state

Track strategy usage per email. Hard ceiling: 4 uses per strategy per batch.

## Foundation Presentation Formats

Rotate between these. Use each at least twice per batch. Never use the same format
for both foundations in one email.

**Format A:** "[Foundation] has given $X to [sector] in [state], with a typical grant
around $Y."

**Format B:** "[Foundation] recently gave $Y to [comparable org] for [purpose]."

**Format C:** "[Foundation] funds [sector] organizations in [state]. Their typical grant
is around $Y."

**Format D:** "The [Foundation] has been funding [sector] work in [state] since [year],
most recently giving $Y to [comparable org]."

## The Offer

What we're offering is a curated set of foundation research, not a raw data dump.

**NAMING:** Use "research," "breakdown," or "foundations," never "list."
- "a breakdown of [display_text] [sector] in [state]"
- "[display_count] foundations that match [org_name]'s profile"
- "the full [display_count]-foundation research"

**"FREE" RULE:** NEVER say "for free," "free list," or "at no cost." Just offer it.
The absence of a price implies it's free.

**CTA ROTATION** (max 4 uses each per batch of 20):
- "Want me to send it?"
- "Interested?"
- "Worth a look?"
- "Just reply and I'll send it over."
- No explicit CTA (let the information stand alone)

**NEVER USE:** "Happy to send it over" / "Let me know if it'd be useful"

## Signature Block

Every email ends with:

Alec Kleinman
TheGrantScout
[rotate one-liner: "Foundation funding research for nonprofits." /
"We research foundation giving patterns for nonprofits." /
"We help nonprofits find foundations that match their work."]

Reply STOP to opt out.

## Tone

The correct register: a peer in your sector who has access to data you might find
useful, sharing it in a short, straightforward email. Professional but not formal.
Specific but not showy. Like someone you met briefly at a conference who followed
up with something relevant.

Use "I" in the body (personal voice). "We" only in the signature line.

## Banned Phrases

**PRESUMPTUOUS:** "to help" / "you should know about" / "you might not know" /
"worth a look" (max 1x per batch) / "worth knowing" / "caught my eye" (max 1x) /
"on your radar"

**SUSPICION TRIGGERS:** "for free" / "free list" / "leads" / "list" (in subject lines)

**TOO CASUAL:** "Happy to send it over" / "Hey [Name]"

**FILLER:** "These days" / "It's been a tough year" / "After everything happening" /
"The past year has been tough" / "navigating uncertainty"

**AI TELLS:** "I came across your organization" / "I was impressed by" / Starting with
"I" / Three adjectives in a row / Generic mission paraphrasing / "I wanted to reach
out because"

**SALES:** "leads" / "prospects" / "top 200 list" / "We compiled" / "We put together"

**CONDESCENDING:** Any sentence implying they don't know something obvious /
Any sentence implying they need saving

## Structural Variation Requirements

In a batch of 20 emails:
- Minimum 4 different opening strategies
- Minimum 4 different foundation presentation formats (A/B/C/D)
- Minimum 4 different CTAs
- Minimum 3 different paragraph counts (2, 3, or 4 paragraphs)
- No transition phrase used more than once ("stood out," "worth a look," etc.)
- Maximum 3 emails following the same structural skeleton

After drafting all 20, re-read them back-to-back. If the template is identifiable
by email 3, rewrite the batch with more structural variation.

## Accuracy Rules (NON-NEGOTIABLE)

1. Every factual claim must trace to a field in the input data. Do not infer,
   round beyond recognition, or add context not in the data.
2. Verify `contact_first_name` plausibly matches `contact_email`. Flag mismatches.
   If `email_quality_flags` contains 'placeholder_name', use "Hi there" greeting.
3. Verify each foundation's `geo_tier` is "primary" or "secondary" for the prospect's state.
4. Verify example grant recipients do similar work to the prospect (check
   `example_grant_purpose` and `example_match_tier`).
5. Verify subject line sector matches body sector. Use `sector_label` consistently.
6. Use the foundation's giving in the prospect's state (from `total_giving_5yr`
   in the cohort row), not nationally.

## Pre-Generation Data Quality Checks

Before generating any emails, run these checks and output a report:
1. Contact name vs. email address consistency (flag `email_quality_flags`)
2. No `email_quality_tier = 'D'` prospects in the batch
3. No `.gov` emails in the batch
4. Geographic tier of all foundations cited (must be primary/secondary/tertiary)
5. `example_match_tier` for each foundation (adjust language per tier)
Flag any prospect that fails a check. Move to manual review queue.

## Output Format

For each email, output:
- **Metadata:** To, Contact, Org, State, Sector, email_quality_tier, Foundations cited
- **Subject line**
- **Email body**
- **Signature block**
- **Internal notes:** Opening strategy used (#1-7), Foundation format used (A/B/C/D),
  CTA used, example_match_tier for each foundation, Any quality flags

At batch end, output:
- Strategy distribution table (strategy # vs. count)
- Foundation format distribution
- CTA distribution
- Paragraph count distribution
- Any data quality flags from pre-generation checks
