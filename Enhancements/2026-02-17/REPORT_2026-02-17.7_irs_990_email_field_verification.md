## Executive Summary

A comprehensive scan of 3,825,764 XML files across all 52 TEOS ZIPs on disk confirms:

- **Filer EMailAddressTxt: 0 instances.** The IRS strips filer email addresses from all public 990 XML releases. This field is a dead end for email enrichment.
- **RecipientEmailAddressTxt: 34,687 instances.** This 990-PF Part XV field (grant application contact email) is populated in ~5.6% of private foundation filings.
- **Database extraction is complete.** Our parser already captures all RecipientEmailAddressTxt values into `pf_returns.app_contact_email`. After filtering junk values (N/A, NONE, etc.), we have **8,638 distinct foundations** with real application contact emails.

**Bottom line:** There is no untapped email data in the IRS 990 XML files. The filer email hypothesis is definitively closed. Our existing RecipientEmailAddressTxt extraction is working correctly.

## Background

Agent 4's enrichment research identified that the IRS 990 XML schema defines an `EMailAddressTxt` field in the Return Header Filer section. If populated in public releases, this could yield 200K-400K foundation and nonprofit emails for free, eliminating the need for expensive enrichment tools.

Our 990-PF parser (`parse_990pf.py`, line 137) already attempts to extract this field but finds 0 values. Our 990/990-EZ parsers don't attempt extraction at all. This investigation aimed to definitively verify whether the field is truly absent or if we have a parsing bug.

## Methodology

### Approach: Byte-Level Scan (Not XML Parsing)

Rather than parsing XML (slow, namespace-sensitive), the verification script uses byte-level string matching on raw XML content. This is:

- **Fast:** 3.8M files in ~28 minutes
- **Reliable:** No false negatives from namespace issues
- **Comprehensive:** Catches the element regardless of XML structure

### Patterns Searched

| Pattern | What It Catches |
|---------|----------------|
| `EMailAddressTxt` | Filer email (the field we hoped existed) |
| `RecipientEmailAddressTxt` | Grant application contact email (990-PF Part XV) |
| `EmailAddress` | Any other email-related XML element |
| `ProfileEmailAddressChangeInd` | IRS metadata flag (not useful for contacts) |

### Filer Email Disambiguation

Files containing `RecipientEmailAddressTxt` also match `EMailAddressTxt` (substring). The script handles this by stripping `RecipientEmailAddressTxt` from the content before checking for standalone `EMailAddressTxt` matches.

**Verdict:** The IRS definitively strips the `EMailAddressTxt` field from all publicly released 990 XML files. This is consistent across all tax years (2016-2024) and all form types (990, 990-EZ, 990-PF). The field exists in the schema definition but is redacted from TEOS releases, similar to SSN/TIN treatment.

### Finding 2: RecipientEmailAddressTxt Is Available (990-PF Only)

| Metric | Count |
|--------|-------|
| XML files with `RecipientEmailAddressTxt` | 34,687 |
| Percentage of all XMLs | 0.91% |
| Percentage of 990-PF filings (est. ~140K/year) | ~5.6% |

This field appears in the 990-PF Part XV Supplementary Information section, where foundations describe how to apply for grants. It represents the email address of the person who handles grant applications.

### Finding 3: Other Email Elements

| Element | Count | Useful? |
|---------|-------|---------|
| `EmailAddress` (any match) | 37,574 | Superset of RecipientEmailAddressTxt + a few others |
| `ProfileEmailAddressChangeInd` | 2,887 | No, IRS metadata flag only |

The `EmailAddress` catch-all count (37,574) vs `RecipientEmailAddressTxt` (34,687) suggests ~2,887 additional matches are `ProfileEmailAddressChangeInd` elements, which is just a boolean flag indicating the filer changed their email on file with the IRS.
