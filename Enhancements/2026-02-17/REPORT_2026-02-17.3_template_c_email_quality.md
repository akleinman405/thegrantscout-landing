## Executive Summary

Reviewed and fixed all 1,236 Template C cold emails in the email campaign CSV. Template C uses the format "I saw that {org_name} {mission_short} and wanted to see if we can help you find private foundation funding." The `mission_short` field must be a third-person singular verb phrase that reads naturally in this sentence.

Starting from ~40% error rate, the final result is **100% pass rate** on all automated quality checks (preambles, comma-and ambiguity, trailing truncation, short entries, org-name leakage). Manual random sampling of 50 emails confirmed natural-sounding quality.

## Problem Statement

Mission_shorts were extracted automatically from IRS 990 mission descriptions (ALL CAPS text). Common problems included:

- **Preamble leakage:** "our mission is to...", "we provide...", "a nonprofit organization that..."
- **Verb agreement errors:** "preserves and protect" instead of "preserves and protects"
- **Truncation artifacts:** Sentences cut mid-phrase, ending on prepositions or adjectives
- **Comma-and ambiguity:** Lists ending with ", X, and Y" creating confusion with the "and wanted to see..." template continuation
- **Org name in text:** The organization's name appearing redundantly in the mission_short
- **Lowercase state names:** "new york" instead of "New York"

## Approach

Combined two automated cleanup functions with 628 manual overrides across 7 rounds of iterative fix-scan-sample cycles.

**Philosophy:** Automated functions for systematic patterns, manual overrides only where automation fails or produces poor results. Each round followed: add fixes, re-run script, re-scan, random sample, identify new patterns, repeat.

## Automated Functions Added

### 1. `fix_verb_agreement(text)`
Fixes bare infinitive verbs after "and" when the sentence starts with a conjugated 3ps verb. Uses a VERB_ONLY set (~50 words) to avoid false positives on ambiguous noun/verb words (e.g., "support", "help", "care").

**Example:** "preserves and protect wildlife" -> "preserves and protects wildlife"

### 2. `capitalize_states(text)`
Capitalizes US state names using regex word boundaries. Covers all 50 states, DC, Puerto Rico, and multi-word entries like "New York City".

**Example:** "serves families in new york" -> "serves families in New York"

### 3. Step 2c cleanup (in `extract_mission_short`)
Post-preamble stripping that removes leading punctuation, org suffixes (INC/LLC), and remaining "is a/an" patterns after org name stripping.

## Notes

- **Organization name truncation:** A few org names in the CSV are truncated (e.g., "Court Appointed Special Advocates of New" instead of "...of New Jersey"). This is a data quality issue in the `organization_name` field, not the `mission_short` field. Should be addressed separately if these emails are sent.
- **Remaining risk:** If new Template C emails are generated in the future, the automated extraction may produce issues for new EINs not covered by manual overrides. The automated functions catch the most common patterns, but preamble stripping and comma-and issues may require new overrides.
- **Override ratio:** 628 overrides / 1,236 Template C = 50.8%. This reflects the difficulty of extracting clean verb phrases from unstructured IRS mission text. The overrides are a necessary investment for email quality.
