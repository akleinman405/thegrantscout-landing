## Phase 3: Manual Foundation Review

**Time:** ~30 min per client
**Purpose:** Human-in-the-loop quality check before enrichment work

### Review Checklist (for top 20 per client)

For each foundation in `02_scored_foundations.csv`:

| Check | Question |
|-------|----------|
| **Sector Fit** | Does their grant history show relevant funding? |
| **Geography Fit** | Do they fund client's state/region? |
| **Size Fit** | Does typical grant match client's needs ($25K-$100K)? |
| **Activity** | Have they given grants in past 2-3 years? |
| **Red Flags** | Known to be invite-only? Dormant? Family foundation? |

### Decision Categories

- **RESEARCH** - Strong fit, worth enrichment research
- **MAYBE** - Decent fit, research if time allows
- **SKIP** - Poor fit, exclude from further consideration

### Target
- Select **15 foundations per client** for enrichment (yields 5 after filtering)

## Phase 4: Enrichment Research (Manual)

**Skill:** `SKILL_foundation_scraper.md`
**Time:** ~20-30 min per foundation (but cached for future clients)
**Purpose:** Gather application process, contacts, deadlines

### For Each Selected Foundation

1. **Find website** (Google: "[Foundation Name] grants" or check 990 URL field)
2. **Locate pages:** Grants, Guidelines, About, Staff/Contact, News/RFPs
3. **Extract fields:**

| Field | What to Capture |
|-------|-----------------|
| `accepts_unsolicited` | TRUE/FALSE - can unknowns apply? |
| `application_type` | open / loi / rfp_periodic / rfp_rolling / invite_only / unknown |
| `application_url` | Direct link to apply |
| `current_deadline` | Next deadline (or "Rolling" / "None stated") |
| `contact_name` | Program officer name |
| `contact_email` | Direct email |
| `contact_phone` | Phone number |
| `program_priorities` | What they fund (quoted from site) |
| `geographic_focus` | Geographic restrictions |
| `grant_range_stated` | Stated grant size range |
| `application_requirements` | What they ask for (LOI, full proposal, etc.) |
| `red_flags` | Array of warnings |

### Red Flag Categories

```
no_unsolicited           - "Do not send unsolicited proposals"
invite_only_strict       - "Grants by invitation only"
no_public_process        - No website or application info
possibly_dormant         - Website old, no recent grants
mission_mismatch         - Priorities don't align
geographic_mismatch      - Excludes client's geography
budget_mismatch          - Only funds different-sized orgs
```

| 3 | **Geographic alignment** (funds client's state) |
| 4 | **Active recent giving** (grants in 2023-2024) |
| 5 | **Appropriate grant size** (median matches client ask) |

### Mix Strategy

- **3 READY foundations** - Clear path to apply
- **1-2 CONDITIONAL** - Good fit, needs verification
- **0-1 WATCH** - Stretch goal, relationship-building only

- 2-3 foundations for **vocational training/disability programs**
- 2-3 foundations for **capital/facility funding**

### Batch Efficiency

- **Scoring (Phase 1)** can run in parallel for all 5 clients
- **Enrichment research (Phase 4)** can be batched - same foundation may serve multiple clients
- **Report generation (Phase 6)** can run sequentially in one batch

### Optimistic Scenario (High Cache Hits)

If most foundations already have enrichment data:
- 5 clients × 1 hour = **5 hours total**

### Conservative Scenario (Low Cache Hits)

If most foundations need fresh research:
- 5 clients × 4 hours = **20 hours total** (spread over 2-3 days)

## Recommended Approach

### Batch Strategy

1. **Day 1: Run all scoring + enrichment checks**
   - Identify total enrichment research needed
   - Prioritize foundations appearing for multiple clients

2. **Day 1-2: Enrichment research sprint**
   - Focus on highest-scoring foundations first
   - Cache results benefit all 5 clients

3. **Day 2-3: Final selection + report generation**
   - Human review of final 5 per client
   - Batch run scripts 04-09
   - Final QC pass

### Parallelization

