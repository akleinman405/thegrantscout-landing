## Changes Made

### Fix 1: Strip script/style tags (ROOT CAUSE fix)

Added `_clean_html()` static method to `NonprofitEmailScraper` that uses BeautifulSoup to decompose `<script>`, `<style>`, and `<noscript>` tags before the email regex runs. This prevents extraction of font designer emails, JS library author emails, and tracking code embedded in CSS/JS bundles.

Called in `_find_all_emails()` after unicode escape handling, before `EMAIL_PATTERN.findall()`.

### Fix 2: Extended EXCLUDE_PATTERNS (+23 entries)

| Category | Count | Examples |
|----------|-------|---------|
| Font metadata domains | 13 | latofonts.com, broofa.com, indiantypefoundry.com |
| Placeholder emails | 7 | john@doe.com, name@email.com, your@email.com |
| Platform vendors | 2 | @givelively.org, program.intake@usda.gov |
| Google Calendar | 1 | @group.calendar.google.com |

### Fix 3: Email type classification

Added `classify_email_type()` function with four categories:

| Type | Count | Description |
|------|-------|-------------|
| personal | 9,216 | Person-name patterns on org domains (john.smith@nonprofit.org) |
| role | 5,466 | Organizational function (info@, contact@, grants@, director@) |
| generic | 2,282 | Free-mail providers (gmail, yahoo, etc.) |
| junk | 21 | Font metadata, placeholders, probers that survived other filters |

### Fix 4: Cross-org dedup (`clean` command)

New CLI command: `python3 cli.py clean [--execute] [--min-orgs N]`

Identifies and deletes emails appearing in N+ distinct organizations (default 3). Dry-run by default. Removed 66 distinct junk emails totaling 701 rows.

### Fix 5: MX validation (`validate-mx` command)

New CLI command: `python3 cli.py validate-mx [--concurrency N]`

Async DNS MX lookups with configurable concurrency (default 50). Validated 8,537 unique domains in 24 seconds.

| MX Status | Emails | % |
|-----------|--------|---|
| Valid | 16,669 | 98.1% |
| Invalid | 316 | 1.9% |

### Fix 6: Enhanced `status` command

Now shows email_type distribution and MX validation breakdown alongside existing metrics.
