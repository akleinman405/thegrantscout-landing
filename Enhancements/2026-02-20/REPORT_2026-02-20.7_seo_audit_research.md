## Site Structure Audit

### robots.txt Analysis

The robots.txt has three conflicting sections:

**Section 1 - Cloudflare Managed:**
- Disallows: Amazonbot, Applebot-Extended, Bytespider, CCBot, ClaudeBot, Google-Extended, GPTBot, meta-externalagent
- Uses non-standard `Content-Signal` directives

**Section 2 - Custom Rules:**
- `User-agent: *` with `Allow: /`, `Disallow: /api/, /_next/, /_vercel/`
- `Crawl-delay: 1`

**Section 3 - AI Crawler Allow:**
- Explicitly allows: GPTBot, ChatGPT-User, ClaudeBot, Claude-User, Claude-SearchBot, PerplexityBot, Google-Extended, Bingbot, FacebookBot

**Problem:** Section 1 disallows ClaudeBot/GPTBot/Google-Extended while Section 3 allows them. Two `User-agent: *` blocks exist. This causes PageSpeed to flag robots.txt as invalid (score 0/100).

**Fix:** Either disable Cloudflare's managed robots.txt injection or remove the custom section and manage everything through Cloudflare's dashboard.

## Core Web Vitals Reference

### Current Thresholds (2025-2026)

| Metric | What It Measures | Good | Needs Improvement | Poor |
|--------|-----------------|:----:|:-----------------:|:----:|
| LCP | Perceived load speed (largest visible element) | ≤ 2.5s | 2.5s - 4.0s | > 4.0s |
| INP | Responsiveness (interaction to visual response) | ≤ 200ms | 200ms - 500ms | > 500ms |
| CLS | Visual stability (unexpected layout movement) | ≤ 0.1 | 0.1 - 0.25 | > 0.25 |

**INP replaced FID on March 12, 2024.** FID only measured the first interaction delay; INP measures all interactions at the 98th percentile.

**Assessment rule:** A page "passes" when 75% of visits meet the "Good" threshold for all three metrics.

### Next.js-Specific Fixes

| Metric | Common Causes | Fix |
|--------|---------------|-----|
| LCP | Large hero images, slow TTFB, render-blocking JS | `next/image` with `priority`, SSG/ISR, minimize client rendering |
| INP | Heavy JS execution, hydration overhead | React Server Components, reduce client JS, defer non-critical interactivity |
| CLS | Images without dimensions, dynamic content, font loading | `width`/`height` on images, `next/font`, avoid layout-shifting dynamic content |

### FAQPage Schema Note

### Google Search Console Verification on Vercel
Recommended method: DNS TXT record.
1. In GSC, choose "Domain" property, enter `thegrantscout.com`
2. Copy the `google-site-verification=...` TXT record
