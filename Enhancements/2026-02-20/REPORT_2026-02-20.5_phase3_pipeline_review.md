## Executive Summary

Phase 3 of the database audit traced the full data flow from IRS XML import through client matching to report generation. The most critical finding is that **the primary matching pipeline (`process_client.py`) is broken** because it depends on the `emb_nonprofit_missions` table, which was dropped on 2026-01-04 to save 52GB of disk space. Additionally, the LASSO V6.1 scorer (`02_score_foundations_v6.1.py`) depends on `calc_foundation_avg_embedding`, which also does not exist.

Key findings:

- **Pipeline is BROKEN.** Both `process_client.py` and the V6.1 scorer reference dropped embedding tables. No client can be processed through the automated pipeline.
- **Only 1 of 8 clients has been fully processed** through the pipeline (PSMF/Patient Safety Movement Foundation, EIN 462730379). It is the only client with LASSO scores, recommendations, proof-of-fit grants, and sibling funders.
- **7 of 8 clients have partial data** -- they have siblings and foundation scores from `process_client.py` but no LASSO scores, no recommendations, and no enrichment.
- **Data quality is strong** for core tables: grant purpose coverage is 100% in fact_grants, recipient EIN coverage is 56.1%, and the foundation profiles table covers all 143,184 foundations.
- **Email pipeline is nascent** -- nonprofit email coverage is 0.9%, foundation email coverage is 6.1%. The email quality tier system exists but 667,009 of 673,381 nonprofits have no tier assigned.
- **No orphaned data** -- all calc table records trace back to valid dim_clients entries.
- **Zero functions/triggers** related to prospect confidence scoring exist in the database.

## Matching Pipeline Trace

### process_client.py (Current Pipeline)

**Location:** `/Users/aleckleinman/Documents/TheGrantScout/4. Pipeline/process_client.py`

This is a single-file, monolithic pipeline that processes one client at a time. It imports `sentence_transformers` and `psycopg2` directly.

#### Phase-by-Phase Breakdown

| Phase | Step | Description | Reads From | Writes To |
|-------|------|-------------|------------|-----------|
| Setup | [1] | Load SentenceTransformer model (`all-MiniLM-L6-v2`) | -- | In-memory |
| Setup | -- | Load client from dim_clients by EIN | `dim_clients` | -- |
| Cleanup | -- | DELETE existing data for this client | -- | `calc_client_foundation_scores`, `calc_client_sibling_grants`, `calc_client_siblings` |
| 3.1 | [2] | Find siblings via embedding similarity | **`emb_nonprofit_missions`** (DROPPED), `nonprofit_returns` | `calc_client_siblings` |
| 3.2.1 | [3] | Populate sibling grants | `fact_grants`, `calc_client_siblings` | `calc_client_sibling_grants` |
| 3.2.2 | [4] | Compute semantic similarity (grant purposes vs target) | `calc_client_sibling_grants` | `calc_client_sibling_grants.semantic_similarity` |
| 3.2.3 | [5] | Keyword matching on grant purposes | `calc_client_sibling_grants` | `calc_client_sibling_grants.keyword_match` |
| 3.2.4 | [6] | Set target grant flags (keyword OR semantic >= 0.55) | `calc_client_sibling_grants` | `calc_client_sibling_grants.is_target_grant`, `target_grant_reason` |
| 3.3 | [7] | Aggregate foundation scores | `calc_client_sibling_grants`, `dim_foundations`, `pf_returns` | `calc_client_foundation_scores` |
| 3.4 | [8] | Enrich with openness/eligibility | `pf_returns` | `calc_client_foundation_scores.open_to_applicants`, `geo_eligible`, `client_eligible` |

**CRITICAL DEPENDENCY:** Phase 3.1 queries `f990_2025.emb_nonprofit_missions` using pgvector's `<=>` (cosine distance) operator. This table was dropped on 2026-01-04. The pipeline will fail with a "relation does not exist" error at step [2].

**Note:** The pipeline sets `lasso_score = NULL` in the INSERT (line 289). LASSO scores are never computed by this script -- they must be computed separately by the V6.1 scorer.
